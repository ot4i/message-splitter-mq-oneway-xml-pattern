Constraints on the use of the Message Splitter for WebSphere MQ: one-way (for XML) pattern

A number of constraints apply when you use this pattern.

You can use the Message Splitter for WebSphere MQ: one-way (for XML) pattern only if you are able to meet all of the following constraints:

  • Your requester and provider applications use WebSphere MQ as the transport mechanism.
  • The incoming messages are well formed XML; that is, XML that parses but does not necessarily validate.
  • The messages conform to the expected structure. Each input message contains:
    • An enclosing envelope, the name of which is optionally identified by a pattern parameter
    • A series of repeated structures, with names optionally identified by a pattern parameter
  • The incoming message is processed automatically in either of the following ways:
    • All of the elements are written to either the output or the bad message queue
    • No elements are written and the message is rolled back
  • You can deploy and run the application that is generated from the pattern instance to an integration node that has a queue manager. To check whether your integration node has a queue manager, in the Integration Nodes view, select the integration node in the Properties view and check that the Queue manager specified on the integration node field is not blank. If the Properties view is not visible, click Window > Show Properties.To specify a queue manager for an integration node, in the Integration Nodes view, right-click the target integration node and click Change, then enter the required queue manager name.

It is assumed that this pattern is for distribution of information that must not be lost. The message flow therefore runs as a transaction and uses persistent messages, which has a significant impact on performance but is necessary because no loss of data is a requirement.

It is assumed that the number of repeated elements is less than the maximum number of uncommitted messages allowed on the output queue. It might be necessary to adjust this limit if large numbers of repeats are anticipated.
