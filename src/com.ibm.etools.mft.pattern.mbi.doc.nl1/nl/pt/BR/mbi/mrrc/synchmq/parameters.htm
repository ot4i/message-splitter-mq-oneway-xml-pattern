Parâmetros para o padrão de Message Correlator para WebSphere MQ: Pedido-Resposta sem Persistência

Informações de entrada
Parâmetro de padrão M C Default Descrição
Fila de entrada Sim Sim IN Este parâmetro padrão identifica a fila do broker na qual pedidos do cliente são recebidos.
Validação de mensagens de entrada Não Sim Nenhum Este parâmetro do padrão identifica o nível de validação necessário para mensagens de entrada. Os valores válidos são:
  • Nenhum
  • Conteúdo
  • Conteúdo e Valor
Se a validação for configurada como Nenhum, a mensagem será tratada como um BLOB opaco.

Se a validação for especificada como Conteúdo ou Conteúdo e Valor, informações de entrada adicionais serão necessárias, as quais são usadas para configurar o nó MQInput.

Esta propriedade é configurável no arquivo BAR. É possível configurar esta propriedade para desativar a validação.

Você não deve usar esta propriedade para ativar a validação, a menos que os parâmetros da mensagem de pedido tenham sido definidos.
Tipo de dados de entrada Não Não XML Este parâmetro do padrão será necessário apenas se o valor de Validação de Mensagens de Entrada for Conteúdo ou Conteúdo e Valor. Os valores válidos são:
  • XML
  • Dados binários (por exemplo, estruturas C ou COBOL)
  • Dados de texto (por exemplo, CSV, SWIFT ou HL7)
Conjunto de mensagens de entrada Não Não   Este parâmetro do padrão será necessário apenas se o valor de Validação de Mensagens de Entrada for Conteúdo ou Conteúdo e Valor.
Tipo de mensagem de entrada Não Não   Este parâmetro do padrão será necessário apenas se o valor de Validação de Mensagens de Entrada for Conteúdo ou Conteúdo e Valor e o valor de Tipo de Dados do Pedido for Dados Binários ou Dados de Texto.
Formato da mensagem de entrada Não Não   Este parâmetro do padrão será necessário apenas se o valor de Validação de Mensagens de Entrada for Conteúdo ou Conteúdo e Valor e o valor de Tipo de Dados de Entrada for Dados Binários ou Dados de Texto.
Informações do Provedor
Parâmetro de padrão M C Default Descrição
Gerenciador de filas de pedidos do provedor Não Sim

Este parâmetro define o gerenciador de filas para o qual todos os pedidos para o provedor são enviados.
Fila de pedidos do provedor Sim Sim PROVIDER

Este parâmetro define a fila para a qual as mensagens de pedido para o provedor são enviadas.
Tempo máximo de resposta em décimos de um segundo Sim Sim Sem tempo limite

Este parâmetro padrão define o tempo máximo a aguardar por uma resposta.

O valor-padrão é Sem tempo limite; entretanto, é possível inserir um número inteiro positivo como o valor para o período de tempo limite no mesmo campo. O período de tempo limite é medido em décimos de um segundo e as mensagens que contêm cabeçalhos MQMD salvos são salvas com este tempo de expiração.

Se o valor for Sem tempo limite, as respostas que são retornadas ao broker serão sempre retornadas ao cliente.

Este parâmetro padrão fornece manutenção automática para as mensagens que são deixadas na fila, mas evita que respostas sejam correspondidas com um cliente após o período de tempo ter decorrido.

Fila de armazenamento Sim Sim STORE

este parâmetro padrão define a fila que é usada para armazenar cabeçalhos MQMD enquanto aguarda uma resposta do provedor.

Esta fila é local para o broker.

  • Nenhum

Esta propriedade é configurável no arquivo BAR. É possível configurar esta propriedade para desativar a validação.

Você não deve usar esta propriedade para ativar a validação, a menos que os parâmetros de mensagem de resposta tenham sido definidos.
Tipo de dado de resposta Não Não XML

Os valores válidos são:
  • XML
  • Dados binários (por exemplo, estruturas C ou COBOL)
  • Dados de texto (por exemplo, CSV, SWIFT ou HL7)

Criação de log
Parâmetro de padrão M C Padrão Descrição
Criação de log requerida Não Não Limpo

Este parâmetro padrão determina se a instância padrão inclui o código para criação de log.

Se Criação de log requerida estiver limpo, o subfluxo de Log não será incluído.

Se Criação de log requerida estiver selecionado, um subfluxo de Log será incluído no fluxo de mensagens e os valores deverão ser configurados para Gerenciador de filas de log e Fila de Logs.
Gerenciador de filas de log Não Sim

Este parâmetro padrão define o gerenciador de filas para criação de log. Ele poderá ser deixado em branco se o gerenciador de filas do broker for usado para criação de log.

Este parâmetro é necessário apenas se Criação de log requerida é selecionado.

Fila de Logs Não Sim LOG

Este parâmetro padrão define a fila para criação de log.

Este parâmetro é necessário apenas se Criação de log requerida é selecionado.
Tratamento de erro
Parâmetro de padrão M C Default Descrição
Mensagem de erro necessária Não Não Selecionado

Este parâmetro padrão define se o aplicativo padrão cria os elementos de fluxo de mensagens para criar mensagens de erro.

Se Mensagem de erro requerida for selecionado, os valores deverão ser configurados para Gerenciador de filas de erros e Fila de erros.
Gerenciador de filas de erros Não Sim  

Este parâmetro padrão define o gerenciador de filas para mensagens de erro. Ele poderá ser deixado em branco se o gerenciador de filas do broker for usado para criação de log.
Fila de erros Não Sim ERROR

Este parâmetro padrão define a fila para mensagens de erro.

Este parâmetro é necessário apenas se Mensagem de erro requerida é selecionado.

Gerais
Parâmetro de padrão M C Default Descrição
Esquema do broker Não Não mqsi

Se um esquema do broker é especificado, todos os fluxos de mensagens e arquivos ESQL são criados nesse esquema. Se nenhum esquema for especificado, o esquema do broker padrão será usado.
Prefixo do fluxo Não Não

Se configurado, este prefixo é incluído no início de todos os nomes de fluxo de mensagens.
Sufixo do fluxo Não Não

Se configurado, este sufixo é incluído no final de todos os nomes de fluxo de mensagens.
Prefixo de Fila Não Não

Se configurado, este prefixo é incluído no início de todos os nomes da fila.

Se os nomes de fila padrão forem usados, o prefixo, o sufixo, ou ambos, deverão ser configurados para identificar nomes de fila para cada instância padrão.

Sufixo da Fila Não Não

Se configurado, este sufixo é incluído no final de todos os nomes de fila.

Se os nomes de fila padrão forem usados, o prefixo, o sufixo, ou ambos, deverão ser configurados para identificar nomes de fila para cada instância padrão.

Descrição resumida Não Não

Esta descrição é incluída na propriedade Descrição Resumida de todos os fluxos de mensagens que são gerados.
Descrição detalhada Não Não

Esta descrição é incluída na propriedade Descrição Detalhada de todos os fluxos de mensagens que são gerados.

Todos os parâmetros padrão são listados com seus valores-padrão.

A coluna intitulada O indica se a propriedade é obrigatória. A coluna intitulada C indica se a propriedade é configurável, significando que é possível alterar o valor quando você inclui o fluxo de mensagens no arquivo broker archive (BAR).

Onde nenhum padrão é indicado, o valor-padrão é uma cadeia vazia.
