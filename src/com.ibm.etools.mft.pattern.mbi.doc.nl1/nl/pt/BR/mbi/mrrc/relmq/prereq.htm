Tarefas para Concluir Antes de Aplicar o Padrão do Message Correlator para WebSphere MQ: Pedido-Resposta com Persistência

Antes de aplicar o padrão para produzir uma instância padrão, é necessário determinar quais das tarefas a seguir são necessárias. Se necessário, conclua as tarefas a seguir:

  • Selecione um nome exclusivo para a instância padrão. É possível usar o padrão várias vezes e criar múltiplas instâncias diferentes que podem ser distinguidas por seus nomes exclusivos. Identifique o propósito da instância padrão para que você possa decidir sobre um nome exclusivo apropriado ou significativo.
  • Selecione os gerenciadores de filas e filas do WebSphere MQ que você deseja que a instância padrão use, por exemplo, para acessar o aplicativo do provedor. Se você estiver usando filas existentes, elas devem ser registradas. Se você desejar criar novos gerenciadores de filas e filas, deverá decidir sobre os nomes para estes recursos.
  • Identifique quaisquer gerenciadores de filas e filas adicionais que sejam necessários para este padrão e decida sobre os nomes exclusivos que você deseja usar. Leve em conta quaisquer políticas, normas ou convenções de nomenclatura que devem ser aplicadas.
  • Decida se a validação das mensagens de pedidos recebidos ou as mensagens de resposta do provedor são necessárias. Se seus aplicativos do solicitante e do provedor gerarem mensagens corretas, a validação poderá não ser necessária. Se você desejar validar suas mensagens, deverá já ter um conjunto de mensagens para fornecer as definições de mensagem ou deverá criar um conjunto de mensagens para fornecer as definições de mensagem.
  • Decida se você requer a criação de log das mensagens de pedido, mensagens de resposta, ou ambas. Se a criação de log for necessária, é preciso decidir para qual gerenciador de filas e fila as mensagens de criação de log são enviadas. É possível enviar mensagens de criação de log para uma fila de log geral que é usada por múltiplos fluxos de mensagens ou você pode usar uma fila especificamente para a instância padrão.
  • Decida se você deseja produzir mensagens de erro para a instância padrão. Se esta instância padrão tratar atualizações, use a manipulação de erros para assegurar que você não perca nenhum dado. É possível enviar mensagens de erro para uma fila de erros geral que é usada por múltiplos fluxos de mensagens ou você pode usar uma fila especificamente para a instância padrão. Se escolher enviar mensagens de erro na instância padrão, é possível ativar esta função ou desativá-la configurando o arquivo broker archive (BAR) antes da implementação.
