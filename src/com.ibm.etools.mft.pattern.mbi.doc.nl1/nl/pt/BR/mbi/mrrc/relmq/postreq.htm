Tarefas para Concluir Após Gerar o Padrão de Message Correlator para WebSphere MQ: Pedido-Resposta com Persistência

As tarefas que você deve concluir após ter gerado o padrão.

  1. Crie o arquivo broker archive (BAR). Inclua o fluxo de mensagens gerado em seu arquivo BAR. No editor do Broker Archive, você deve selecionar a opção Compilar e Sequenciar Recursos antes de construir o arquivo BAR. Os conjuntos de mensagens que são referenciados pelo fluxo também devem ser inclusos.
  2. Configure o arquivo broker archive. A configuração adicional do fluxo de mensagens no arquivo bar não é necessária antes de implementar os fluxos de mensagens para sua instância padrão do Message Correlator para WebSphere MQ: Pedido-Resposta com Persistência, mas talvez você deseje revisar algumas das opções padrão e configurar alterações para seu ambiente.
  3. Crie filas. Quando aplicar o padrão, uma lista de filas do WebSphere MQ é gerada, com nomes de filas e nomes do gerenciador de filas, que devem ser criados se ainda não existirem. Todas as configurações de mensagem que são necessárias para o padrão são configuradas explicitamente nas mensagens e não são dependentes das configurações de filas.
    1. No arquivo de resumo, visualize a lista de filas que você deve criar. O arquivo de resumo está localizado na pasta Configuração de Padrão do projeto de instância padrão InstanceName.
    2. Crie recursos do WebSphere MQ. É possível criar recursos do WebSphere MQ na linha de comandos usando programas e utilitários como runmqsc ou usando a interface gráfica do WebSphere MQ Explorer. Para obter informações adicionais sobre a criação de recursos do WebSphere MQ, consulte WebSphere MQ Intercommunication, disponível na página da Web da biblioteca do WebSphere MQ.
    3. Configure as filas de entrada e de resposta para ter filas de restauração para preservação de mensagens quando erros causarem a restauração do fluxo de mensagens.
  4. Implemente sua abordagem monitorando e corrigindo erros. Isto pode requerer o monitoramento da fila para mensagens de erro ou o monitoramento do log de eventos e das filas de restauração.
