Tarefas para Concluir Após Gerar o Padrão de Message Splitter para WebSphere MQ: Unidirecional (para XML)

As tarefas que você deve concluir após ter gerado o padrão.

  1. Crie o arquivo broker archive (BAR). O fluxo de mensagens deve ser incluído em um arquivo bar. Qualquer conjunto de mensagens que é referenciado pelo fluxo é incluído automaticamente.
    1. Clique em Arquivo > Novo > Arquivo BAR.
    2. Digite o nome do projeto do servidor ou selecione da lista exibida. A lista é filtrada para mostrar apenas projetos no conjunto de trabalhos ativo.
    3. Insira um nome para o arquivo BAR sendo criado.
    4. Clique em Concluir. É criado um arquivo com uma extensão .bar e exibido na visualização do Navegador de Administração do Intermediário, sob a pasta Broker Archives. O editor de Conteúdo para o arquivo BAR é aberto.
    5. Na guia Preparar, selecione o arquivo de fluxo de mensagens gerado para incluí-lo em seu arquivo BAR.
    6. Selecione Compilar e Sequenciar Recursos e, em seguida, clique em Construir e Salvar para criar seu arquivo BAR.
  2. Configure o arquivo broker archive. A configuração adicional do fluxo de mensagens no arquivo bar não é necessária antes de implementar os fluxos de mensagens para sua instância padrão de Message Splitter para WebSphere MQ: Unidirecional (para XML), mas talvez você deseje revisar algumas das opções opção e configurar alterações para o seu ambiente.

    Na página Gerenciar do editor de Broker Archive:

    1. Opcional: Use a propriedade Criação de log requerida para selecionar se a criação de log está ativada ou desativada para o fluxo de mensagens do Message Splitter.
    2. Opcional: Use a mensagem Mensagem de erro requerida para selecionar se mensagens de erro são geradas para o fluxo de mensagens do Message Splitter.
    3. Opcional: Também é possível alterar qualquer uma das propriedades configuráveis no broker archive, por exemplo, propriedades para segurança ou criar instâncias adicionais. É necessário assegurar que qualquer valor alterado não cause impacto no comportamento da instância padrão.
  3. Crie filas. Quando aplicar o padrão, uma lista de filas do WebSphere MQ é gerada, com nomes de filas e nomes do gerenciador de filas, que devem ser criados se ainda não existirem. Todas as configurações de mensagem que são necessárias para o padrão são configuradas explicitamente nas mensagens e não são dependentes das configurações de filas.
    1. No arquivo de resumo, visualize a lista de filas que você deve criar. O arquivo de resumo está localizado na pasta Configuração de Padrão do projeto de instância padrão InstanceName.
    2. Crie recursos do WebSphere MQ. É possível criar recursos do WebSphere MQ na linha de comandos, usando programas e utilitários como runmqsc, ou usando a interface gráfica do WebSphere MQ Explorer. Para obter informações adicionais sobre a criação de recursos do WebSphere MQ, consulte WebSphere MQ Intercommunication, disponível na página da Web da biblioteca do WebSphere MQ.
    3. Se você usar o valor Rotas de Consulta no parâmetro Roteamento, as rotas poderão ser definidas dinamicamente e não serão listadas no arquivo de resumo. Quando rotas são incluídas no banco de dados, as filas correspondentes devem ser criadas.
  4. Configure um banco de dados de roteamento se necessário. Se o valor Rotas de Consulta for selecionado no parâmetro Roteamento, a origem de dados deverá ser configurada com uma tabela ROUTING_TABLE. Esta tabela deve incluir as seguintes colunas:
    • ROUTING_CONTEXT

      Use um valor que identifica as entradas de roteamento para este fluxo.

    • VARIABLE1

      O valor ao qual a propriedade KEY1 do elemento de mensagem deve corresponder.

    • VARIABLE2

      O valor ao qual a propriedade KEY2 do elemento de mensagem deve corresponder.

    • QUEUE_MANAGER

      O gerenciador de filas do destino.

    • QUEUE

      A fila de destino.
