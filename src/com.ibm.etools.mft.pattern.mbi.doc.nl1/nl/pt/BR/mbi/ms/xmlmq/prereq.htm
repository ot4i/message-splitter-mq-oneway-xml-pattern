Tarefas para Concluir Antes de Aplicar o Padrão de Message Splitter para WebSphere MQ: Unidirecional (para XML)

Antes de aplicar o padrão para produzir uma instância padrão, é necessário determinar quais das tarefas a seguir são necessárias. Se necessário, conclua as tarefas a seguir:

  • Selecione um nome exclusivo para a instância padrão. É possível usar o padrão várias vezes e criar várias instâncias diferentes, que podem ser distinguidas por seus nomes exclusivos. Identifique o propósito do aplicativo padrão para que você possa decidir sobre um nome exclusivo apropriado ou significativo.
  • Selecione os gerenciadores de filas e filas do WebSphere MQ que você deseja que a instância padrão use, por exemplo, para receber a mensagem XML. Se você estiver usando filas existentes, elas devem ser registradas. Se você desejar criar novos gerenciadores de filas e filas, deverá decidir sobre os nomes para estes recursos.
  • Decida se a validação é necessária e defina um conjunto de mensagens ou determine o nome do conjunto de mensagens. Após o padrão ter sido gerado, conclua a tarefa de criação do conjunto de mensagens.
  • Identifique quaisquer gerenciadores de filas e fila adicionais que sejam necessários para este padrão e decida sobre os nomes que você deseja usar. Leve em conta quaisquer políticas, normas ou convenções de nomenclatura que devem ser aplicadas.
  • Decida se você requer a criação de log de mensagens recebidas. Se a criação de log for necessária, é preciso decidir para qual gerenciador de filas e fila as mensagens de criação de log são enviadas. É possível enviar mensagens de criação de log para uma fila de log geral que é usada por múltiplos fluxos de mensagens ou você pode usar uma fila especificamente para a instância padrão.
  • Decida se há múltiplos destinos possíveis e, se houver, selecione a abordagem de roteamento a usar.
    • Configure o parâmetro Roteamento como Especificar rotas, se você tiver um número limitado de rotas em que os destinos não são alterados frequentemente.
    • Configure o parâmetro Roteamento como Consultar rotas, se o conjunto de rotas de destino não estiver totalmente determinado ou puder ser alterado frequentemente. Se este valor for selecionado, o nome da origem de dados deverá ser conhecido.
  • Decida a abordagem de manipulação de erros. Decida se você deseja elementos de mensagem que não podem ser roteados corretamente para causar uma reversão completa ou se é suficiente gravar o elemento com falha em uma fila para processamento da exceção.
  • Decida se você deseja produzir mensagens de erro para a instância padrão. Todos os dados são preservados sem mensagens de erro porque as ações de erro preservam elementos em uma fila para este propósito ou revertem o fluxo de mensagens. A mensagem recebida é retornada para a fila recebida ou é enviada para uma fila de restauração. Entretanto, se estas filas não forem monitoradas, mensagens de erro poderão ser requeridas como um mecanismo de notificação.

    Se escolher enviar mensagens de erro na instância padrão, é possível ativar esta função ou desativá-la configurando o arquivo broker archive (BAR) antes da implementação.
