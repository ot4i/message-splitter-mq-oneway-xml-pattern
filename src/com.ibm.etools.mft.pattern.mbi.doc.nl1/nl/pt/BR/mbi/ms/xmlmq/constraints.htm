Restrições no Uso do Padrão de Message Splitter para WebSphere MQ: Unidirecional (para XML)

Várias restrições se aplicam quando este padrão é usado.

É possível usar o padrão de Message Splitter para WebSphere MQ: Unidirecional (para XML) apenas se você puder atender a todas as restrições a seguir:

  • Seus aplicativos de solicitante e do provedor usam o WebSphere MQ como o mecanismo de transporte.
  • As mensagens recebidas são XML bem-formado; ou seja, XML que analisa mas não necessariamente valida.
  • As mensagens estão em conformidade com a estrutura esperada. Cada mensagem de entrada contém:
    • Um envelope anexo, com o nome pelo qual é identificado opcionalmente por um parâmetro padrão
    • Uma série de estruturas repetidas, com nomes identificardos opcionalmente por um parâmetro padrão
  • A mensagem recebida é processada automaticamente das seguintes maneiras:
    • Todos os elementos são gravados na saída ou na fila de mensagens inválidas
    • Nenhum elemento é gravado e a mensagem é revertida

Assume-se que este padrão é para distribuição de informações que não devem ser perdidas. O fluxo de mensagens, portanto, é executado como uma transação e usa mensagens persistentes, o que tem um impacto significativo no desempenho mas é necessário porque nenhuma perda de dados é um requisito.

Assume-se que o número de elementos repetidos é menor que o número máximo de mensagens não confirmadas permitidas na fila de saída. Pode ser necessário ajustar este limite de números grandes de repetições forem antecipados.
