Message Correlator for WebSphere MQ: 지속을 사용하는 요청-응답 패턴의 오류 서브플로우

요청 메시지 플로우 처리 중에 오류가 포착되면, 오류 서브플로우가 호출됩니다.

오류 메시지를 포함하도록 패턴 매개변수를 구성하고 사용자 정의 특성 ErrorLoggingOn이 선택되면, 다음 조치가 수행됩니다.

  1. 요청 처리 중에 오류가 발생하면, 메시지를 성공적으로 송신하지 못했음을 알리는 오류 응답이 클라이언트로 리턴됩니다.

    응답 처리 중에 오류가 발생하면, 응답 주소를 알 수 없기 때문에 오류 응답이 리턴되지 않습니다.

    다음 메시지는 표준 XML 오류 메시지의 예입니다.

    <Error><Status>-1</Status><Text>메시지 송신 실패</Text></Error>
  2. 오류 메시지를 준비하고 오류 큐에 기록됩니다. 예외 세부사항이 XML 메시지로 기록됩니다. 세부사항은 다음과 같습니다.
    • 브로커 이름
    • 플로우 이름
    • 시간소인
    • 예외 데이터 요약
  3. 데이터를 보존하기 위해 메시지 플로우를 롤백합니다.
  4. 오류 감지 및 정정에 필요한 정보가 오류 메시지에 포함되어 있습니다.
