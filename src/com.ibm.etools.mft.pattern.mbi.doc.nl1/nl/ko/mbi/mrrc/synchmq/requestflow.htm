Message Correlator for WebSphere MQ: 지속을 사용하지 않는 요청-응답 패턴의 요청 플로우

요청 플로우는 트랜잭션 플로우로 실행됩니다.

요청 플로우가 수신한 메시지는 다음과 같은 방법으로 처리됩니다.

  1. 메시지 플로우가 메시지를 수신하면 MQMD 헤더에 응답 대상 큐가 있도록 확인합니다. 이 필드가 공백이면 오류가 생성됩니다.
  2. 메시지 플로우가 메시지를 수신하면 상관 ID를 메시지 ID로 설정하여 메시지 MQMD 헤더의 사본을 가져오고 수정합니다. 원본 응답 주소를 보존하기 위해 메시지 헤더의 컨텐츠가 큐에 저장됩니다. 저장된 헤더에 상관 ID를 설정하면 헤더를 검색하고 요청자에 대한 응답에 추가할 때 값이 올바르고 상관 ID에 의한 요청자 검색을 사용할 수 있습니다.
  3. 메시지 플로우는 RequestProcessor라는 서브플로우를 포함하고 있습니다. 이 서브플로우는 Passthrough 노드만 포함하므로 처리가 수행되지 않습니다. 패턴의 구조를 변경하지 않고 고유 처리를 추가하도록 패턴 인스턴스에서 이 서브플로우를 사용자 정의할 수 있습니다.
  4. 수신되는 메시지의 MQMD 헤더를 수정하여 요청자의 응답 대상 큐 관리자와 응답 대상 큐를 응답 메시지 플로우의 입력 노드 주소와 바꿉니다.
  5. 메시지가 제공자 애플리케이션으로 전달됩니다.
  6. 메시지 플로우는 로그라는 로그 기록을 위한 서브플로우를 포함하고 있을 수 있습니다. 로그 서브플로우를 포함하거나 제외하도록 패턴을 구성할 수 있습니다.
  7. 로그 기록이 필요하지 않으면 메시지를 제공자 큐에 기록한 후 메시지 플로우가 성공적으로 종료되고 트랜잭션이 커미트됩니다.
  8. 메시지 플로우는 오류라고 하는 오류 핸들링을 위한 서브플로우도 포함하고 있습니다.
  9. 요청자 애플리케이션 메시지 플로우 또는 제공자 애플리케이션에서 제한시간 초과가 발생하면 비지속 메시지가 제한시간 초과됩니다.
