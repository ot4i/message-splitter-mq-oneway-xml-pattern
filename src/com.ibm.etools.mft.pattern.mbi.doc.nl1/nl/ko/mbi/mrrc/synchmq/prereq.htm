Message Correlator for WebSphere MQ: 지속을 사용하지 않는 요청-응답 패턴을 적용하기 전 완료할 태스크

패턴을 적용하여 패턴 인스턴스를 생성하기 전에 다음 중 필요한 태스크를 판별해야 합니다. 필요하면 다음 태스크를 완료하십시오.

  • 패턴 인스턴스의 고유 이름을 선택하십시오. 패턴을 여러 번 사용하고 고유 이름으로 구분할 수 있는 다양한 인스턴스를 여러 개 작성할 수 있습니다. 적합한 이름이나 의미 있는 고유 이름을 결정할 수 있도록 패턴 인스턴스의 용도를 확인하십시오.
  • 제공자 애플리케이션 액세스와 같은 목적으로 패턴 인스턴스가 사용할 WebSphere MQ 큐 관리자 및 큐를 선택하십시오. 기존 큐를 사용하는 경우 이를 기록해야 합니다. 새 큐 관리자 및 큐를 작성하려면 해당 자원의 이름을 결정해야 합니다.
  • 이 패턴에 필요한 추가 큐 관리자 및 큐를 확인하고 사용할 이름을 결정하십시오. 적용해야 하는 정책, 표준 또는 이름 규칙을 고려하십시오.
  • 수신되는 요청 메시지 또는 제공자 응답 메시지의 유효성 검증이 필요한지 여부를 결정하십시오. 요청자와 제공자 애플리케이션이 올바른 메시지를 생성하면 유효성 검증이 필요하지 않습니다. 메시지의 유효성을 검증하려면 메시지 정의를 제공하기 위한 메시지 세트가 이미 있거나 메시지 정의를 제공하기 위해 메시지 세트를 작성해야 합니다.
  • 요청 메시지나 응답 메시지 또는 둘 다의 로그 기록이 필요한지 여부를 결정하십시오. 로그 기록이 필요하면 로그 기록 메시지를 송신할 큐 관리자 및 큐를 결정해야 합니다. 여러 메시지 플로우에서 사용되는 일반 로그 큐로 로그 기록 메시지를 송신하거나 패턴 인스턴스 고유의 큐를 사용할 수 있습니다.
  • 패턴 인스턴스에 대한 오류 메시지를 생성할 것인지 여부를 결정하십시오. 이 패턴 인스턴스가 업데이트를 핸들링하는 경우에는 데이터가 손실되지 않도록 오류 핸들링을 사용하십시오. 여러 메시지 플로우에서 사용되는 일반 오류 큐로 오류 메시지를 송신하거나 패턴 인스턴스 고유의 큐를 사용할 수 있습니다. 패턴 인스턴스에 오류 메시지를 송신하도록 선택한 경우 배치 전에 브로커 아카이브(BAR) 파일을 구성하여 이 기능을 켜거나 끌 수 있습니다.
