Message Correlator for WebSphere MQ: 지속을 사용하지 않는 요청-응답 패턴의 로그 서브플로우

로그 서브플로우가 요청 플로우에 포함되어 있으면 사용자 정의 특성 RequestLoggingOn에서 요청 로그 기록을 제어합니다. 로그 서브플로우가 요청 플로우에 포함되어 있으면, 사용자 정의 특성 ResponseLoggingOn에서 요청 로그 기록을 제어합니다.

RequestLoggingOn 특성이 선택되면, 로그 기록이 필요하고 로그 서브플로우가 호출되어 플로우를 종료합니다. 로그 기록이 실패하면 플로우에서 예외가 포착되고 정보가 Trace 노드에 기록됩니다.

로그 기록이 응답 플로우에 포함되어 있으면 로그 서브플로우가 호출됩니다. 로그 기록의 모든 오류가 포착되고 플로우가 성공적으로 종료하며 플로우 트랜잭션이 커미트됩니다.

로그 메시지가 WebSphere MQ 큐에 지속 WebSphere MQ 메시지로 기록됩니다. 그런 다음 이 로그 메시지는 조직의 표준 관행에 따라 따로 처리됩니다.
