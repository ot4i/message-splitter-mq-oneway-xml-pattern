Message Splitter for WebSphere MQ: 단방향(XML용) 패턴의 사용 제한조건

이 패턴을 사용할 때는 많은 제한조건이 존재합니다.

다음과 같은 제한조건을 모두 충족하는 경우에만 Message Splitter for WebSphere MQ: 단방향(XML용) 패턴을 사용할 수 있습니다.

  • 요청자 및 제공자 애플리케이션이 WebSphere MQ를 전송 메커니즘으로 사용합니다.
  • 수신되는 메시지가 구분 분석되며 유효성 검증이 불필요한 올바른 형식의 XML이어야 합니다.
  • 메시지가 예상 구조를 준수합니다. 각 입력 메시지는 다음을 포함합니다.
    • 패턴 매개변수에 의해 선택적으로 식별되는 이름의 동봉 인벨로프
    • 패턴 매개변수에 의해 선택적으로 식별되는 이름의 일련의 반복되는 구조
  • 수신되는 메시지는 다음 방법 중 하나를 통해 자동으로 처리됩니다.
    • 모든 요소는 출력 또는 잘못된 메시지 큐에 기록됩니다.
    • 요소가 기록되지 않으며 메시지가 롤백됩니다.

이 패턴은 손실되지 않아야 하는 정보 분배를 위한 것으로 가정합니다. 따라서 메시지 플로우가 트랜잭션으로 실행되고 지속 메시지를 사용합니다. 이 작업은 성능에 상당한 영향을 줄 수 있지만 데이터 손실 방지가 요구사항이므로 필요합니다.

반복되는 요소의 수가 출력 큐에 허용되는 커미트되지 않은 메시지의 최대 수보다 작다고 가정합니다. 많은 반복이 예상되는 경우 이 한계를 조정해야 할 수 있습니다.
