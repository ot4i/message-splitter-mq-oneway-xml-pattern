Message Splitter for WebSphere MQ: 단방향(XML용) 패턴을 적용하기 전 완료할 태스크

패턴을 적용하여 패턴 인스턴스를 생성하기 전에 다음 중 필요한 태스크를 판별해야 합니다. 필요하면 다음 태스크를 완료하십시오.

  • 패턴 인스턴스의 고유 이름을 선택하십시오. 패턴을 여러 번 사용하고 고유 이름으로 구분할 수 있는 다양한 인스턴스를 여러 개 작성할 수 있습니다. 적합한 이름이나 의미 있는 고유 이름을 결정할 수 있도록 패턴 애플리케이션의 용도를 확인하십시오.
  • XML 메시지 수신과 같은 목적으로 패턴 인스턴스가 사용할 WebSphere MQ 큐 관리자 및 큐를 선택하십시오. 기존 큐를 사용하는 경우 이를 기록해야 합니다. 새 큐 관리자 및 큐를 작성하려면 해당 자원의 이름을 결정해야 합니다.
  • 유효성 검증이 필요한지 여부를 결정하고 메시지 세트를 정의하거나 메시지 세트의 이름을 판별하십시오. 패턴을 생성한 후에는 메시지 세트 작성 태스크를 완료하십시오.
  • 이 패턴에 필요한 추가 큐 관리자 및 큐를 확인하고 사용할 이름을 결정하십시오. 적용해야 하는 정책, 표준 또는 이름 규칙을 고려하십시오.
  • 수신되는 메시지의 로그 기록이 필요한지 여부를 결정하십시오. 로그 기록이 필요하면 로그 기록 메시지를 송신할 큐 관리자 및 큐를 결정해야 합니다. 여러 메시지 플로우에서 사용되는 일반 로그 큐로 로그 기록 메시지를 송신하거나 패턴 인스턴스 고유의 큐를 사용할 수 있습니다.
  • 목적지가 여러 개인지 확인하고, 여러 개이면 사용할 라우팅 방법을 선택하십시오.
    • 목적지가 자주 변경되지 않는 제한된 수의 라우트가 있으면, 라우팅 매개변수를 라우트 지정으로 설정하십시오.
    • 대상 라우트를 완전히 판별할 수 없거나 자주 변경되면, 라우팅 매개변수를 라우트 검색으로 설정하십시오. 이 값이 선택되면 데이터 소스의 이름을 알고 있어야 합니다.
  • 오류 핸들링 방법을 결정하십시오. 올바르게 라우팅할 수 없는 메시지 요소가 있으면 전체 롤백되도록 할 것인지 여부 또는 예외 처리를 위해 실패한 요소를 큐에 기록하기에 충분한지 여부를 결정하십시오.
  • 패턴 인스턴스에 대한 오류 메시지를 생성할 것인지 여부를 결정하십시오. 모든 데이터는 오류 메시지 없이 보존됩니다. 오류 조치가 이러한 목적으로 큐에 요소를 보존하거나 메시지 플로우를 롤백하기 때문입니다. 수신되는 메시지가 수신 큐로 리턴되거나 백아웃 큐로 송신됩니다. 그러나 이 큐를 모니터링하지 않는 경우에는 알림 메커니즘으로 오류 메시지가 필요할 수 있습니다.

    패턴 인스턴스에 오류 메시지를 송신하도록 선택한 경우 배치 전에 브로커 아카이브(BAR) 파일을 구성하여 이 기능을 켜거나 끌 수 있습니다.
