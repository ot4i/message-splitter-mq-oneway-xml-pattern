Flujo Response para el patrón Correlacionador de mensajes para WebSphere MQ: solicitud-respuesta sin persistencia

El flujo Response se ejecuta como un flujo no transaccional.

Cuando se recibe una respuesta de la aplicación de proveedor, se efectúan las acciones siguientes:

  1. El flujo de mensajes incluye un subflujo llamado ResponseProcessor. Este subflujo sólo contiene un nodo Passthrough, por lo que no se realiza ningún proceso. Puede personalizar este subflujo en la instancia del patrón para añadir su propio proceso sin cambiar la estructura del patrón.
  2. La cabecera MQMD guardada se recupera de la cola de almacenamiento y se añade al mensaje de respuesta, de forma que el mensaje de respuesta tenga la dirección correcta para responder al solicitante.
  3. La respuesta se devuelve al solicitante.
  4. El flujo de mensajes puede contener un subflujo para realizar el registro denominado Log. Puede configurar el patrón para incluir o excluir el registro. Si se incluye el registro, se llama al subflujo Log.
  5. Si el registro no está incluido, el flujo de mensajes finaliza correctamente y la transacción se confirma.
  6. El flujo de mensajes también contiene un subflujo de manejo de errores, llamado Error.
  7. Los mensajes de error se escriben en la cola de errores si ha configurado esta opción para la instancia de patrón.
