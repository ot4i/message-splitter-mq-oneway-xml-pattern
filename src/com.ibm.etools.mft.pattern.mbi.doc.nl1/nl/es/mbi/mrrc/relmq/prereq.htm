Tareas que deben completarse antes de aplicar el patrón Correlacionador de mensajes para WebSphere MQ: solicitud-respuesta con persistencia

Antes de aplicar el patrón para producir una instancia de patrón, debe determinar cuáles de las tareas siguientes son necesarias. Si es preciso, realice las tareas siguientes:

  • Seleccione un nombre exclusivo para la instancia de patrón. Puede utilizar el patrón varias veces y crear varias instancias distintas, que pueden distinguirse por sus nombres exclusivos. Identifique el fin de la instancia de patrón para que pueda decidir un nombre adecuado o significativo exclusivo.
  • Seleccione las colas y los gestores de colas de WebSphere MQ que desee que utilice la instancia de patrón, por ejemplo, para acceder a la aplicación de proveedor. Si está utilizando colas existentes, deben estar registradas. Si desea crear nuevos gestores de colas y colas, debe tomar una decisión sobre los nombres de estos recursos.
  • Identifique cualquier cola y gestor de colas adicionales que este patrón necesite, y decida qué nombres exclusivos desea utilizar. Tenga en cuenta las políticas, estándares o convenios de denominación que deben aplicarse.
  • Decida si es necesaria la validación de los mensajes de solicitud entrantes o los mensajes de respuesta del proveedor. Si las aplicaciones de solicitante y proveedor generan mensajes correctos, es posible que la validación no sea necesaria. Si desea validar los mensajes, ya debe tener un conjunto de mensajes para proporcionar las definiciones de mensajes, o debe crear un conjunto de mensajes para proporcionar las definiciones de mensajes.
  • Decida si necesita un registro de mensajes de solicitud, de mensajes de respuesta, o ambos. Si el registro es necesario, debe decidir a qué gestor de colas y cola se enviarán los mensajes de registro. Puede enviar mensajes de registro a una cola de registro general que utilicen varios flujos de mensajes, o puede utilizar una cola específicamente para la instancia de patrón.
  • Decida si desea producir mensajes de error para la instancia de patrón. Si esta instancia de patrón maneja actualizaciones, utilice el manejo de errores para asegurarse de no perder datos. Puede enviar mensajes de error a una cola de errores generales que utilizan varios flujos de mensajes, o puede utilizar una cola específicamente para la instancia de patrón. Si elige enviar mensajes de error en la instancia de patrón, puede activar o desactivar esta función configurando el archivo archivador de intermediario (BAR) antes del despliegue.
