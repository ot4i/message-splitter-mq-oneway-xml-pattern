Parámetros para el patrón Correlacionador de mensajes para WebSphere MQ: solicitud-respuesta con persistencia

Información de entrada
Parámetro de patrón O C Predeterminado Descripción
Cola de entrada Sí Sí IN Este parámetro de patrón identifica la cola de intermediario en la que se reciben las solicitudes.
Mensajes de validación de entrada No Sí Ninguno Este parámetro identifica el nivel de validación necesario para los mensajes de entrada. Los valores válidos son:
  • Ninguno
  • Contenido
  • Contenido y valor
Si la validación está establecida en Ninguno, el mensaje se trata como un BLOB opaco.

Si la validación se especifica como Contenido o Contenido y valor, se necesita información de entrada adicional, que se utiliza para configurar el nodo MQInput.

Esta propiedad se puede configurar en el archivo BAR. Puede configurar esta propiedad para desactivar la validación.

No debe utilizar esta propiedad para activar la validación a menos que se hayan definido los parámetros del mensaje de solicitud.
Tipo de datos de entrada No No XML Este parámetro de patrón es necesario sólo si el valor de Validación de entrada es Contenido o Contenido y valor. Los valores válidos son:
  • XML
  • Datos binarios (por ejemplo, estructuras C o COBOL)
  • Datos de texto (por ejemplo, CSV, SWIFT o HL7)
Conjunto de mensajes de entrada No No  

Este parámetro de patrón es necesario sólo si el valor de Validación de entrada es Contenido o Contenido y valor.
Tipo de mensaje de entrada No No  

Este parámetro de patrón solamente es necesario si el valor de Validación de mensajes de entrada es Contenido o Contenido y valor, y si el valor de Tipo de datos de entrada es Datos binarios o Datos de texto.
Formato de mensaje de entrada No No  

Este parámetro de patrón solamente es necesario si el valor de Validación de mensajes de entrada es Contenido o Contenido y valor, y si el valor de Tipo de datos de entrada es Datos binarios o Datos de texto.
Información de proveedor
Parámetro de patrón O C Predeterminado Descripción
Gestor de colas de solicitud de proveedor No Sí   Este parámetro de patrón identifica el gestor de colas en el que la aplicación de proveedor recibe solicitudes del flujo de mensajes de intermediario. Se puede dejar en blanco si se utiliza el gestor de colas de intermediario para el registro.
Cola de solicitud de proveedor Sí Sí PROVIDER Este parámetro de patrón identifica la cola en la que la aplicación de proveedor recibe solicitudes del flujo de mensajes de intermediario.
Tiempo máximo de respuesta en décimas de segundo Sí Sí Sin tiempo de espera Este parámetro de patrón define el tiempo máximo de espera de una respuesta.

El valor predeterminado es Sin tiempo de espera; no obstante, puede especificar un entero positivo como valor para el periodo de tiempo de espera en el mismo campo. El periodo de tiempo de espera se mide en décimas de segundo y los mensajes que contienen cabeceras MQMD guardadas se guardan con este tiempo de caducidad.

Si el valor es Sin tiempo de espera, las respuestas que se devuelven al intermediario se devuelven siempre al cliente.

Este parámetro de patrón proporciona limpieza automática para los mensajes que se dejan en la cola, pero impide que se emparejen las respuestas con un cliente después de que el periodo de tiempo haya transcurrido.
Cola de almacenamiento Sí Sí STORE Este parámetro de patrón define la cola que se utiliza para almacenar cabeceras MQMD mientras esperan una respuesta del proveedor.

Esta cola es local para el intermediario.
Información de respuesta
Parámetro de patrón O C Predeterminado Descripción
Cola de respuestas Sí Sí RESPONSE Este parámetro de patrón identifica la cola de intermediario en la que se reciben las solicitudes.
Validación de mensajes de respuesta No Sí Ninguno Este parámetro identifica el nivel de patrón de validación necesario para los mensajes de respuesta. Los valores válidos son:
  • Ninguno
  • Contenido
  • Contenido y valor

Esta propiedad se puede configurar en el archivo BAR. Puede configurar esta propiedad para desactivar la validación.

No debe utilizar esta propiedad para activar la validación a menos que se hayan definido los parámetros del mensaje de solicitud.
Tipo de datos de respuesta No No XML Este parámetro de patrón es necesario sólo si el valor de Validación de respuesta es Contenido o Contenido y valor. Los valores válidos son:
  • XML
  • Datos binarios (por ejemplo, estructuras C o COBOL)
  • Datos de texto (por ejemplo, CSV, SWIFT o HL7)
Conjunto de mensajes de respuesta No No   Este parámetro de patrón es necesario sólo si el valor de Validación de respuesta es Contenido o Contenido y valor.
Tipo de mensaje de respuesta No No   Este parámetro de patrón solamente es necesario si el valor de Validación de respuesta es Contenido o Contenido y valor, y si Tipo de datos de respuesta es Datos binarios o Datos de texto.
Formato de mensaje de respuesta No No   Este parámetro de patrón solamente es necesario si el valor de Validación de respuesta es Contenido o Contenido y valor, y si Tipo de datos de respuesta es Datos binarios o Datos de texto.
Registro
Parámetro de patrón O C Predeterminado Descripción
Registro obligatorio No No Borrado

Este parámetro de patrón determina si la instancia de patrón incluye el código de registro.

Si se borra Registro necesario, no se incluye un subflujo Log.

Si se selecciona Registro obligatorio, se incluye un subflujo Log en el flujo de mensajes, y se deben establecer los valores para Gestor de colas de registro y Cola de registro.
Gestor de colas de registro No Sí

Este parámetro de patrón define el gestor de colas para el registro. Se puede dejar en blanco si se utiliza el gestor de colas de intermediario para el registro.

Este parámetro es necesario sólo si se ha seleccionado Registro necesario.
Cola de registro No Sí LOG

Este parámetro de patrón define la cola de registro.

Este parámetro es necesario sólo si se ha seleccionado Registro necesario.
Manejo de errores
Parámetro de patrón O C Predeterminado Descripción
Mensaje de error obligatorio No No Seleccionado

Este parámetro de patrón define si la aplicación de patrón crea los elementos de flujo de mensajes que crean los mensajes de error.

Si Mensaje de error necesario está seleccionado, deben establecerse valores para Gestor de colas de errores y Cola de errores.
Gestor de colas de errores No Sí  

Este parámetro de patrón define el gestor de colas para los mensajes de error. Se puede dejar en blanco si se utiliza el gestor de colas de intermediario para el registro.
Cola de errores No Sí ERROR

Este parámetro de patrón define la cola para mensajes de error.

Este parámetro es necesario sólo si se ha seleccionado Mensaje de error necesario.
General
Parámetro de patrón O C Predeterminado Descripción
Esquema de intermediario No No mqsi Si se ha establecido, todos los flujos de mensajes y archivos ESQL se crean en el esquema de intermediario indicado. Si no se ha establecido, se utiliza el esquema predeterminado.
Prefijo de flujo No No   Si se establece, este prefijo se añade al inicio de todos los nombres de flujo de mensajes.
Sufijo de flujo No No   Si se establece, este sufijo se añade al final de todos los nombres de flujo de mensajes.
Prefijo de cola No No   Si se establece, este prefijo se añade al inicio de todos los nombres de cola.

Si se utilizan nombres de cola predeterminados, debe establecerse el prefijo, el sufijo, o ambos para identificar los nombres de cola para cada instancia de patrón.
Sufijo de cola No No   Si se establece, este sufijo se añade al final de todos los nombres de cola.
Descripción corta No No   Esta descripción se añade a la propiedad Descripción corta de todos los flujos de mensajes que se generan.
Descripción larga No No   Esta descripción se añade a la propiedad Descripción larga de todos los flujos de mensajes que se generan.

Todos los parámetros de patrón aparecen en la lista con sus valores predeterminados.

La columna con la cabecera O indica si la propiedad es obligatoria. La columna con la cabecera C indica si la propiedad es configurable, lo que quiere decir que puede cambiar el valor al añadir el flujo de mensajes al archivo archivador de intermediario (BAR).

Si no se indica el valor predeterminado, éste es una serie vacía.
