Restricciones en el uso del patrón Divisor de mensajes para WebSphere MQ: unidireccional (para XML)

Se aplican una serie de restricciones cuando se utiliza este patrón.

Puede utilizar el patrón Divisor de mensajes para WebSphere MQ: unidireccional (para XML) sólo si puede cumplir todas las restricciones siguientes:

  • Las aplicaciones de solicitante y proveedor utilizan WebSphere MQ como mecanismo de transporte.
  • Los mensajes de entrada están en XML correcto; es decir, XML que analiza pero no necesariamente valida.
  • Los mensajes se ajustan a la estructura esperada. Cada mensaje de entrada contiene:
    • Un sobre cuyo nombre se identifica opcionalmente mediante un parámetro de patrón
    • Una serie de estructuras repetidas, cuyos nombres se identifican opcionalmente mediante un parámetro de patrón
  • El mensaje de entrada se procesa automáticamente de una de estas formas:
    • Todos los elementos se graban en la salida o bien en la cola de mensajes erróneos
    • No se graba ningún elemento, y se retrotrae el mensaje

Se supone que este patrón es para la distribución de información que no se ha perdido. Por consiguiente, el flujo de mensajes se ejecuta como una transacción y utiliza mensajes persistentes que influyen de forma significativa en el rendimiento, pero es necesario porque es un requisito que no haya pérdida de datos.

Se supone que el número de elementos repetidos es inferior al número máximo de mensajes no comprometidos permitidos en la cola de salida. Puede ser necesario ajustar este límite si se ha previsto un gran número de repeticiones.
