Untergeordneter Protokollnachrichtenfluss für das Muster Message Splitter for WebSphere MQ: one-way (for XML) (Nachrichtenverteiler für WebSphere MQ: unidirektional (bei XML))

Protokollnachrichten werden als persistente WebSphere MQ-Nachrichten in eine WebSphere MQ-Warteschlange geschrieben.

Diese Protokollnachrichten können unabhängig voneinander gemäß der in Ihrem Unternehmen implementierten Standardverfahren verarbeitet werden. Das Nachrichtenprotokoll enthält die ursprüngliche Nachricht sowie die folgenden Informationen in einem MQRFH2-Header:

  • Zeitmarke
  • Brokername
  • Nachrichtenflussname
  • Status (0 für Erfolg)
