Vorgaben für die Verwendung des Musters Message Splitter for WebSphere MQ: one-way (for XML) (Nachrichtenverteiler für WebSphere MQ: unidirektional (bei XML))

Wenn Sie dieses Muster verwenden, gelten eine Reihe von Vorgaben.

Sie können das Muster Message Splitter for WebSphere MQ: one-way (for XML) (Nachrichtenverteiler für WebSphere MQ: unidirektional (bei XML)) nur verwenden, wenn alle der folgenden Vorgaben erfüllt sind:

  • Ihre Requester- und Provideranwendungen verwenden WebSphere MQ als Transportmechanismus.
  • Die eingehenden Nachrichten bestehen aus richtig formatiertem XML-Code, d. h. XML-Code, der syntaktisch analysiert, jedoch nicht unbedingt ausgewertet werden kann.
  • Die Nachrichten entsprechen der erwarteten Struktur. Jede Eingabenachricht enthält Folgendes:
    • Eine umschließende Rahmenanweisung, deren Name optional durch einen Musterparameter identifiziert wird
    • Eine Reihe sich wiederholender Strukturen, deren Namen optional durch einen Musterparameter identifiziert werden
  • Die eingehende Nachricht wird auf eine der folgenden Arten automatisch verarbeitet:
    • Alle Elemente werden in die Ausgabewarteschlange oder in die Warteschlange für fehlerhafte Nachrichten geschrieben
    • Keine Elemente werden geschrieben und die Nachricht wird zurückgesetzt

Es wird vorausgesetzt, dass dieses Muster der Verteilung von Informationen dienen soll, die nicht verloren gehen dürfen. Der Nachrichtenfluss wird deshalb als Transaktion ausgeführt und verwendet persistente Nachrichten, was beträchtliche Auswirkungen auf die Leistung hat, jedoch notwendig ist, da die Vermeidung von Datenverlust eine Voraussetzung ist.

Es wird vorausgesetzt, dass die Anzahl der sich wiederholenden Elemente kleiner ist als die maximal zulässige Anzahl nicht festgeschriebener Nachrichten in der Ausgabewarteschlange. Dieser Grenzwert muss möglicherweise angepasst werden, wenn eine große Anzahl von Wiederholungen erwartet wird.
