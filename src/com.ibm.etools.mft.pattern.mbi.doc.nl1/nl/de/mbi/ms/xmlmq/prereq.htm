Tasks, die vor der Anwendung des Musters Message Splitter for WebSphere MQ: one-way (for XML) (Nachrichtenverteiler für WebSphere MQ: unidirektional (bei XML)) ausgeführt werden müssen

Bevor Sie das Muster für die Erstellung einer Musterinstanz anwenden, müssen Sie bestimmen, welche der folgenden Tasks erforderlich sind. Führen Sie folgende Tasks aus, falls erforderlich:

  • Wählen Sie einen eindeutigen Namen für die Musterinstanz aus. Sie können das Muster mehrmals verwenden und verschiedene Instanzen davon erstellen, die sich durch ihre eindeutigen Namen unterscheiden. Legen Sie einen aussagekräftigen eindeutigen Namen fest, der auf den Zweck der Musteranwendung hinweist.
  • Wählen Sie die Warteschlangenmanager und Warteschlangen von WebSphere MQ aus, die die Musterinstanz zum Beispiel für den Empfang von XML-Nachrichten verwenden soll. Wenn Sie vorhandene Warteschlangen verwenden, müssen diese erfasst werden. Wenn Sie neue WS-Manager und Warteschlangen erstellen möchten, müssen Sie für diese Ressourcen Namen festlegen.
  • Entscheiden Sie, ob eine Validierung erforderlich ist, und definieren Sie einen Nachrichtensatz oder legen Sie den Namen für den Nachrichtensatz fest. Beenden Sie die Task für die Nachrichtensatzerstellung, nachdem das Muster generiert wurde.
  • Ermitteln Sie alle weiteren für dieses Muster erforderlichen Warteschlangenmanager und Warteschlangen und überlegen Sie sich geeignete Namen. Berücksichtigen Sie alle Richtlinien, Standards oder Namenskonventionen, die beachtet werden müssen.
  • Entscheiden Sie, ob eine Protokollierung der eingehenden Nachrichten erforderlich ist. Falls ja, bestimmen Sie den Warteschlangenmanager und die Warteschlange für die Protokollnachrichten. Die Protokollnachrichten können an eine allgemeine Protokollwarteschlange gesendet werden, die von mehreren Nachrichtenflüssen verwendet wird, oder an eine speziell für die Musterinstanz eingerichtete Warteschlange.
  • Ermitteln Sie, ob mehrere mögliche Ziele vorliegen, und entscheiden Sie sich in diesem Fall für eine Routing-Lösung.
    • Setzen Sie den Parameter Routing auf Routen angeben, wenn nur eine eingeschränkte Anzahl an Routen vorhanden ist und sich die Ziele nur selten ändern.
    • Setzen Sie den Parameter Routing auf Routen suchen, wenn Sie den Satz der Zielrouten nicht genau bestimmen können oder sich die Zielrouten häufig ändern. Bei dieser Einstellung muss der Name der Datenquelle bekannt sein.
  • Bestimmen Sie die Fehlerbehandlungsmethode. Entscheiden Sie, ob Nachrichtenelemente, die nicht korrekt weitergeleitet werden können, zu einem vollständigen Rollback führen sollen, oder ob es ausreicht, das fehlgeschlagene Element an eine Warteschlange zur Ausnahmeverarbeitung weiterzuleiten.
  • Entscheiden Sie, ob Sie Fehlernachrichten für die Musterinstanz erstellen möchten. Auch ohne Fehlernachrichten bleiben alle Daten erhalten, da Nachrichtenelemente bei einem Fehler durch die Fehlerbehandlungsaktionen entweder in eine spezielle Warteschlange für diesen Zweck eingereiht werden oder der Nachrichtenfluss zurückgesetzt wird. Die eingehende Nachricht wird entweder an die Eingangswarteschlange zurückgesendet oder an eine Rücksetzwarteschlange weitergeleitet. Falls diese Warteschlangen jedoch nicht ständig überwacht werden, empfiehlt es sich, Fehlernachrichten als Benachrichtigungsmechanismus zu aktivieren.

    Wenn Sie Fehlernachrichten in der Musterinstanz senden möchten, können Sie diese Funktion aktivieren oder inaktivieren, indem Sie die Brokerarchivdatei (BAR-Datei) vor der Implementierung konfigurieren.
