生成 Message Correlator for WebSphere MQ: request-response without persistence 模式后要完成的任务

生成该模式之后必须完成的任务。

  1. 创建代理归档 (BAR) 文件。将生成的消息流添加到 BAR 文件中。在“代理归档”编辑器中，必须先选中编译并内联资源选项，然后再构建 BAR 文件。另外，还必须添加该流所引用的消息集。
  2. 配置代理归档文件。为 Message Correlator for WebSphere MQ: request-response without persistence 模式实例部署消息流之前，不需要在代理归档文件中进一步配置消息流，但您可能要针对您的环境复审一些缺省选项并配置更改。
  3. 创建队列。应用模式时，将生成一个 WebSphere MQ 队列的列表，其中会列出队列名称和队列管理器名称；如果不存在这些 WebSphere MQ 队列，就必须加以创建。模式所需的所有消息设置都必须明确设置在消息上，且独立于队列的设置。
    1. 在摘要文件中，查看您必须创建的队列列表。摘要文件位于 InstanceName 模式实例项目的模式配置文件夹中。
    2. 创建 WebSphere MQ 资源。 您可以使用程序和实用程序（如 runmqsc）在命令行上创建 WebSphere MQ 资源，也可以使用 WebSphere MQ Explorer 图形界面来完成创建。有关创建 WebSphere MQ 资源的更多信息，请参阅 WebSphere MQ Library Web 页面上提供的 WebSphere MQ Intercommunication。
  4. 如果配置了错误消息来提供错误通知，请监视并清除错误队列。
