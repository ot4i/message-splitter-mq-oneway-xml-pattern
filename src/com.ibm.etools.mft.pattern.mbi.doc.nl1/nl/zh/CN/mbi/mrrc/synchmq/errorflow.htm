Message Correlator for WebSphere MQ: request-response without persistence 模式的错误子流

如果在请求消息流处理期间捕捉到错误，那么将调用错误子流。

如果已配置模式参数包含错误消息，并且选择了用户定义的属性 ErrorLoggingOn，那么将需要执行以下操作：

  1. 如果在请求处理期间发生错误，会将错误响应返回至客户机，指示消息发送失败。

    如果在响应处理期间发生错误，将不会返回错误响应，因为应答地址未知。

    以下消息是标准 XML 错误消息示例：

    <Error><Status>-1</Status><Text>Failed to send message</Text></Error>
  2. 准备错误消息并将其写入错误队列。写入的异常详细信息将作为 XML 消息。详细信息包括：
    • 代理名称
    • 流名称
    • 时间戳记
    • 异常数据的摘要
