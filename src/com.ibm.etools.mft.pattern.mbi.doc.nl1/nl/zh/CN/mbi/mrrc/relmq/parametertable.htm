Message Correlator for WebSphere MQ: request-response with persistence 模式的参数

输入信息
模式参数 M C 缺省值 描述
输入队列 是 是 IN 此模式参数标识从中接收客户机请求的代理队列。
输入消息验证 否 是 无 此模式参数标识输入消息所需的验证级别。 有效值为：
  • 无
  • 内容
  • 内容和值
如果验证设置为无，那么消息将作为不透明 BLOB。

如果验证指定为内容或内容和值，那么需要更多输入信息以用于配置 MQInput 节点。

此属性可在 BAR 文件中配置。 您可以配置此属性来关闭验证。

除非已经定义了请求消息参数，否则不能使用此属性开启验证。
输入数据类型 否 否 XML 仅当输入消息验证的值为内容或内容和值时，才需要此模式参数。 有效值为：
  • XML
  • 二进制数据（例如，C 或 COBOL 结构）
  • 文本数据（例如，CSV、SWIFT 或 HL7）
输入消息集 否 否  

仅当输入消息验证的值为内容或内容和值时，才需要此模式参数。
输入消息类型 否 否  

仅当输入消息验证的值为内容或内容和值，并且输入数据类型的值为二进制数据或文本数据时，才需要此模式参数。
输入消息格式 否 否  

仅当输入消息验证的值为内容或内容和值，并且输入数据类型的值为二进制数据或文本数据时，才需要此模式参数。
提供者信息
模式参数 M C 缺省值 描述
提供者请求队列管理器 否 是   此模式参数标识提供者应用程序从代理消息流接收请求时所使用的队列管理器。 如果将代理队列管理器用于记录，那么此参数可保留为空。
提供者请求队列 是 是 PROVIDER 此模式参数标识提供者应用程序从代理消息流接收请求时所使用的队列。
最大响应时间（以 100 毫秒为单位） 是 是 无超时 此模式参数定义等待响应的最长时间。

缺省值为无超时；但您可以在同一字段中输入一个正整数作为超时周期值。超时周期以 100 毫秒为单位计算，在到期后将保存持有已保存 MQMD 头的消息。

如果值为无超时，那么返回至代理的响应将总是返回到客户机。

此模式参数为队列中的剩余消息提供自动仓库管理，但在到期后阻止将响应匹配到客户机。

存储队列 是 是 STORE 此模式参数定义在等待提供者响应时用于存储 MQMD 头的队列。

此队列位于代理本地。

响应信息
模式参数 M C 缺省值 描述
响应队列 是 是 RESPONSE 此模式参数标识从中接收客户机请求的代理队列。
响应消息验证 否 是 无 此模式参数标识响应消息所需的验证级别。有效值为：
  • 无
  • 内容
  • 内容和值

此属性可在 BAR 文件中配置。 您可以配置此属性来关闭验证。

除非已经定义了请求消息参数，否则不能使用此属性开启验证。
响应数据类型 否 否 XML 仅当“响应验证”的值为“内容”或“内容和值”时，才需要此模式参数。 有效值为：
  • XML
  • 二进制数据（例如，C 或 COBOL 结构）
  • 文本数据（例如，CSV、SWIFT 或 HL7）
响应消息集 否 否   仅当“响应验证”的值为“内容”或“内容和值”时，才需要此模式参数。
响应消息类型 否 否   仅当“响应验证”的值为“内容”或“内容和值”，并且“响应数据类型”为“二进制数据”或“文本数据”时，才需要此模式参数。
响应消息格式 否 否   仅当“响应验证”的值为“内容”或“内容和值”，并且“响应数据类型”为“二进制数据”或“文本数据”时，才需要此模式参数。
记录
模式参数 M C 缺省值 描述
需要记录 否 否 已清除

此模式参数确定模式实例是否包括用于记录的代码。

如果清除了需要记录，那么将不包括日志子流。

如果选择了需要记录，那么消息流中将包括日志子流，并且必须设置日志队列管理器和日志队列的值。
日志队列管理器 否 是

此模式参数定义用于记录的队列管理器。 如果将代理队列管理器用于记录，那么此参数可保留为空。

仅当选择了需要记录时才需要此参数。
日志队列 否 是 LOG

此模式参数定义用于记录的队列。

仅当选择了需要记录时才需要此参数。
错误处理
模式参数 M C 缺省值 描述
需要错误消息 否 否 已选中

此模式参数定义模式应用程序是否创建消息流元素，该元素创建错误消息。

如果选择了需要错误消息，那么必须设置错误队列管理器和错误队列的值。
错误队列管理器 否 是  

此模式参数定义错误消息的队列管理器。 如果将代理队列管理器用于记录，那么此参数可保留为空。
错误队列 否 是 ERROR

此模式参数定义错误消息的队列。

仅当选择了需要错误消息时才需要此参数。

常规
模式参数 M C 缺省值 描述
代理模式 否 否 mqsi 如果设置了此参数，那么将在指定的代理模式中创建所有的消息流和 ESQL 文件。如果未设置此参数，那么将使用缺省模式。
流前缀 否 否   如果设置了此参数，那么会将此前缀添加到所有消息流名称的开始处。
流后缀 否 否   如果设置了此参数，那么会将此后缀添加到所有消息流名称的结尾处。
队列前缀 否 否   如果设置了此参数，那么会将此前缀添加到所有队列名称的开始处。

如果使用了缺省队列名称，那么必须设置前缀和/或后缀，以标识每个模式实例的队列名称。

队列后缀 否 否   如果设置了此参数，那么会将此后缀添加到所有队列名称的结尾处。
简短描述 否 否   此描述将被添加到生成的所有消息流的简短描述属性。
详细描述 否 否   此描述将被添加到生成的所有消息流的详细描述属性。

列出所有模式参数及其缺省值。

标题为 M 的列指示属性是否为必需的。 标题为 C 的列指示属性是否为可配置的，也就是说，可以在将消息流添加到代理归档 (BAR) 文件时更改值。

未表明缺省值时，缺省值是空字符串。
