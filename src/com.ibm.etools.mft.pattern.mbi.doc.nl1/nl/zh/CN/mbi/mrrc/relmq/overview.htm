Message Correlator for WebSphere MQ: request-response with persistence 模式

使用 Message Correlator for WebSphere MQ: request-response with persistence 模式可在单个队列上接受来自多个客户机应用程序的请求，并使用事务流和持久 WebSphere MQ 消息将响应返回到正确的客户机。

因为此模式使用带有持久消息和事务流的可靠通信，所以在消息交换导致要求不损失数据的更新时适合使用该模式。

解决方案

解决方案将实现一个调解流，它会先将每个通过消息标识建立索引的请求消息中的 MQMD 头信息存储起来，然后再将带有所有头的消息转发给提供者应用程序。这一针对提供者的请求消息在消息头的应答部分中指定代理的响应队列。根据常规做法，提供者应用程序应将消息标识从请求消息复制到响应消息的相关标识字段。收到响应消息时，代理流使用相关标识来检索先前存储的 MQMD 头，并使用应答地址来更新入局 MQMD 头。此操作可确保使用的是请求者的原始回复地址。然后，带有所有头的响应消息将返回到请求者。
