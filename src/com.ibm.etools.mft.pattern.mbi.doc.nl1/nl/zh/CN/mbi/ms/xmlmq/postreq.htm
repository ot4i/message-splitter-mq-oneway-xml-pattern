生成 Message Splitter for WebSphere MQ: one-way (for XML) 模式之后要完成的任务

生成该模式之后必须完成的任务。

  1. 创建代理归档 (BAR) 文件。必须将消息流添加到代理归档文件中。将自动添加该流引用的任何消息集。
    1. 单击“文件”>“新建”>“BAR 文件”。
    2. 输入服务器项目的名称，或者从显示的列表中选择一个名称。此列表已过滤，仅会显示活动工作集中的项目。
    3. 为您要创建的 BAR 文件输入一个名称。
    4. 单击“完成”。将创建一个扩展名为 .bar 的文件，该文件显示在“代理管理导航器”视图中的“代理归档”文件夹下。此时将为 BAR 文件打开“内容”编辑器。
    5. 在“准备”选项卡上，选择生成的消息流文件，以便将其添加到 BAR 文件中。
    6. 选择编译并内联资源，然后单击构建并保存以创建 BAR 文件。
  2. 配置代理归档文件。为 Message Splitter for WebSphere MQ: one-way (for XML) 模式实例部署消息流之前，不需要在代理归档文件中进一步配置消息流，但应当针对您的环境复审一些缺省选项并配置更改。

    在“代理归档”编辑器的“管理”页面中：

    1. 可选：使用需要记录属性来选择是要为消息分离器消息流开启还是关闭记录。
    2. 可选：使用需要错误消息属性来选择是否要为消息分离器消息流生成错误消息。
    3. 可选：您还可以更改代理归档中的任何可配置属性，例如，安全性属性或用于创建其他实例的属性。您必须确保更改的任何值都不会影响模式实例的行为。
  3. 创建队列。应用模式时，将生成一个 WebSphere MQ 队列的列表，其中会列出队列名称和队列管理器名称；如果不存在这些 WebSphere MQ 队列，就必须加以创建。模式所需的所有消息设置都必须明确设置在消息上，且独立于队列的设置。
    1. 在摘要文件中，查看您必须创建的队列列表。摘要文件位于 InstanceName 模式实例项目的模式配置文件夹中。
    2. 创建 WebSphere MQ 资源。您可以使用程序和实用程序（如 runmqsc）在命令行上创建 WebSphere MQ 资源，也可以使用 WebSphere MQ Explorer 图形界面来完成创建。有关创建 WebSphere MQ 资源的更多信息，请参阅 WebSphere MQ Library Web 页面上提供的 WebSphere MQ Intercommunication。
    3. 如果在路由参数中使用查找路由值，那么可以在数据库中动态定义路由，但路由不会在摘要文件中列出。在将路由添加到数据库时，必须创建对应的队列。
  4. 如果需要，请设置路由数据库。如果在路由参数中选择了查找路由值，那么必须用 ROUTING_TABLE 表来设置数据源。此表必须包含以下列：
    • ROUTING_CONTEXT

      使用一个标识此流路由条目的值。

    • VARIABLE1

      消息元素中的 KEY1 属性必须匹配的值。

    • VARIABLE2

      消息元素中的 KEY2 属性必须匹配的值。

    • QUEUE_MANAGER

      目标的队列管理器。

    • QUEUE

      目标队列。
