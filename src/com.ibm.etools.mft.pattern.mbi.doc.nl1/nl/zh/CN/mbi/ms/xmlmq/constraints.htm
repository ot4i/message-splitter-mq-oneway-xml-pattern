使用 Message Splitter for WebSphere MQ: one-way (for XML) 模式时的约束

使用此模式时会有若干约束。

只有在满足以下所有约束时，才可以使用 Message Splitter for WebSphere MQ: one-way (for XML) 模式：

  • 请求者和提供者应用程序使用 WebSphere MQ 作为传输机制。
  • 入局消息是格式正确的 XML；即，可解析但不必进行验证的 XML。
  • 消息遵循预期的结构。每个输入消息包含：
    • 外层包络，其名称可由模式参数标识
    • 一系列重复的结构，其名称可由模式参数标识
  • 采用以下一种方式自动处理入局消息：
    • 将所有元素写入到输出或不正确消息队列
    • 不写入任何元素并回滚消息

假设该模式要分发的信息不得丢失。因此，消息流作为事务运行并且使用持久消息，这对性能会有非常明显的影响，但却是必需的，因为要求不得丢失数据。

假设重复元素的数目少于输出队列所允许的未落实消息的最大数目。如果预测到重复的数目非常大，那么可能需要调整该限制。
