Message Splitter for WebSphere MQ: one-way (for XML) 模式的参数

输入信息
模式参数 M C 缺省值 描述
输入队列 是 是 IN 该模式参数标识从中接收 XML 消息的代理队列。
容器名称 是 否 *:* 保存要抽取的元素的入局 XML 消息中容器元素的路径。该路径使用类似 ESQL 的语法指定，但是仅包含 XML 中的元素；例如，SalesData.Invoices。

您可以指定完整的名称空间，但除非您需要区分两个名称相同但名称空间不同的容器元素，否则，可以将名称空间指定为通配符；例如，*:SalesData.*:Invoices。

如果使用缺省值 *:*，那么将假定为 XML 消息的根元素，它可以是任何名称。

消息元素名称 是 否 *:* 从入局消息抽取的元素名称。

如果使用缺省值 *:*，那么会接受容器的任何子代。

不必指定名称空间，因为假设为无论名称空间如何，指定名称的任何元素都是必需的。

路由
模式参数 M C 缺省值 描述
路由 否 否 无路由 有效值为：
  • 无路由
  • 指定路由
  • 查找路由
无路由
模式参数 M C 缺省值 描述
输出队列管理器 否 是   该参数定义向其发送所有输出消息的队列管理器。如果保留为空，那么缺省值为代理队列管理器。
输出队列 否 是 OUT 此参数定义目标队列。
指定路由

以下可选名称空间表中的参数定义名称空间前缀和名称空间 URI，您可以使用它们在 XPATH 表达式中定义路由的键位置。如果包含行，那么必须包含这两个参数。您可以添加、编辑和删除值，以及更改值在表中的顺序。

模式参数 M C 缺省值 描述
名称空间前缀 是 否 不适用 名称空间前缀参数包含在路由定义的 XPath 键位置参数中。
名称空间 URI 是 否 不适用 名称空间 URI 是与指定的前缀关联的名称空间。

如果路由参数的值为指定路由，那么以下路由表中的参数定义输出目标和规则。 四个参数可用于定义一个路由，并且可以按照需要重复定义更多的路由。您可以添加、编辑和删除值，以及更改值在表中的顺序。

模式参数 M C 缺省值 描述
键位置 是 否 不适用

键位置是一个 XPath 表达式，该表达式定义正在传播的记录中的某个位置。例如，$Root/XMLNSC/submitPORequest/partNo。

有关要使用的表达式类型的更多信息，请参阅 IBM Integration 文档中的“Route 节点”和“使用 XPath”。

如果名称空间指定于可选的名称空间表，那么可包括这些名称空间。

键值 是 否 不适用 此参数包含的值必须与入局消息中的“键位置”字段匹配，才能选择此路由。
队列管理器 是 是   该参数定义此路由目标的队列管理器。如果保留为空，那么缺省值为代理队列管理器。
队列 是 是 不适用 该参数定义此路由目标的队列。
查找路由
模式参数 M C 缺省值 描述
路由数据源 否 是 不适用 此参数定义可从其中选择路由的数据源。可在 ROUTING_TABLE 表中查找路由信息。
路由上下文 否 否 不适用 该参数定义路由上下文，以便您可以将 ROUTING_TABLE 用于多个调解流。

只有 ROUTING_CONTEXT 字段匹配路由数据上下文值的行才能被选择用来进行路由查找，并添加到内存高速缓存。

键位置 1 否 否 不适用 此参数定义如何从传播的消息元素中抽取第一个键。此参数定义为 ESQL 表达式。

第一个键用于在数据源路由条目中的 VARIABLE1 上进行选择。 位置必须为 InputBody 或 InputRoot 格式。

键位置 2 否 否 不适用 此参数定义如何从传播的消息元素中抽取可选的第二个键。此参数定义为 ESQL 字段引用表达式。

第二个键用于在数据源路由条目中的 VARIABLE2 上进行选择。 位置必须为 InputBody 或 InputRoot 格式。

高速缓存刷新 否 否 已清除 如果选中该参数，那么在收到刷新消息时，将会从数据源刷新路由数据的内存高速缓存。
刷新队列 否 是 REFRESH 如果选中高速缓存刷新，那么发送到该队列的消息会触发内存路由数据刷新。
记录
模式参数 M C 缺省值 描述
需要记录 否 否 已清除 此模式参数确定模式实例是否包括用于记录的代码。

如果已清除，那么不包括记录子流。

如果已选中，那么必须设置日志队列管理器和日志队列的值。

日志队列管理器 否 是   此模式参数定义用于记录的队列管理器。 如果将代理队列管理器用于记录，那么此参数可保留为空。

仅当选择了需要记录时才需要此参数。

日志队列 否 是 LOG 此模式参数定义用于记录的队列。

仅当选择了需要记录时才需要此参数。

错误处理
模式参数 M C 缺省值 描述
需要错误消息 否 是 已选中 此模式参数确定模式实例是否包括用于将错误消息放在队列中的代码。

如果已选中，那么必须设置错误队列管理器和错误队列的值。

错误队列管理器 否 是   此模式参数定义用于错误记录的队列管理器。

仅当选择了需要错误消息时才需要此参数。

错误队列 否 是 ERROR 此模式参数定义错误消息的队列。

仅当选择了需要错误消息时才需要此参数。

不正确消息操作 否 否 保存 该参数定义在无法路由消息元素时所执行的操作。有效值为：
  • 保存
  • 取消

如果选中保存，那么任何无法路由的消息元素都将置于不正确消息队列中。

如果选中取消且无法路由消息元素，那么会回滚整个消息流。

不正确消息队列管理器 否 是   此模式参数定义用于保存无法路由的消息元素的队列管理器。 如果保留为空，那么缺省值为代理队列管理器。

只有在不正确消息操作参数值为保存时，该参数才是必需的。

不正确消息队列 否 是 BAD 此模式参数定义用于保存无法路由的消息元素的队列。

只有在不正确消息操作参数值为保存时，该参数才是必需的。

常规
模式参数 M C 缺省值 描述
代理模式 否 否 mqsi 如果设置了此参数，那么将在指定的代理模式中创建所有的消息流和 ESQL 文件。如果未设置此参数，那么将使用缺省模式。
流前缀 否 否   如果设置了此参数，那么会将此前缀添加到所有消息流名称的开始处。
流后缀 否 否   如果设置了此参数，那么会将此后缀添加到所有消息流名称的结尾处。
队列前缀 否 否   如果设置了此参数，那么会将此前缀添加到所有队列名称的开始处。

如果使用了缺省队列名称，那么必须设置前缀和/或后缀，以标识每个模式实例的队列名称。

队列后缀 否 否   如果设置了此参数，那么会将此后缀添加到所有队列名称的结尾处。
简短描述 否 否   此描述将被添加到生成的所有消息流的简短描述属性。
详细描述 否 否   此描述将被添加到生成的所有消息流的详细描述属性。

列出所有模式参数及其缺省值。

标题为 M 的列指示属性是否为必需的。 标题为 C 的列指示属性是否为可配置的，也就是说，可以在将消息流添加到代理归档 (BAR) 文件时更改值。

未表明缺省值时，缺省值是空字符串。
