Message Splitter for WebSphere MQ: one-way (for XML) 模式

使用 Message Splitter for WebSphere MQ: one-way (for XML) 模式可将较大的 XML 消息拆分为较小元素，以供一个或多个目标通过使用事务流和持久 WebSphere MQ 消息进行处理。

当您的应用程序存储有关多个业务事务的信息并通过批处理的方式将该信息传送至一个或多个目标应用程序时，请使用此模式。它可用于处理大型消息，而不会占用过多内存。

该模式可用于以下情况：源应用程序或目标应用程序无法执行或处理近乎实时的服务调用，或者出于业务的原因而要求批处理信息。

解决方案

解决方案用于实现使用可变消息树的消息流，以便能够在处理可重复元素对其进行删除。部分解析支持根据需求解析单个元素，并且，由于这些元素在完成时会被删除，从而避免了由于元素数量的增长而导致的内存树线性增长。在抽取每个元素时，会将其路由至适当的目标应用程序。
