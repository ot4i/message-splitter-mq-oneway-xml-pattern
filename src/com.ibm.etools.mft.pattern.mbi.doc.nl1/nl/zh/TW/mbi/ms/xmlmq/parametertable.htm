Message Splitter for WebSphere MQ: one-way (for XML) 型樣的參數

輸入資訊
型樣參數 M C 預設值 說明
輸入佇列 是 是 IN 此型樣參數識別在其上接收 XML 訊息的分配管理系統佇列。
儲存器名稱 是 否 *:* 位於送入 XML 訊息的儲存器元素的路徑，而此訊息保留要取出的元素。此路徑是使用 ESQL 這類語法所指定，但是只包括 XML 中的元素；例如，SalesData.Invoices。

您可以完整指定名稱空間，但是，除非您需要區分兩個名稱相同但名稱空間不同的儲存器元素，否則可以使用萬用字元來指定名稱空間；例如，*:SalesData.*:Invoices。

如果使用預設值 *:*，則會採用 XML 訊息的根元素，而它可以是任何名稱。

訊息元素名稱 是 否 *:* 從送入訊息取出的元素名稱。

如果使用預設值 *:*，則會接受儲存器的任何子項。

因為假設需要給定名稱的任何元素（不論名稱空間為何），所以不得指定名稱空間。

型樣參數 M C 預設值 說明
輸出佇列管理程式 否 是   此參數定義要傳送所有輸出訊息的目標佇列管理程式。如果保留為空白，則預設值是分配管理系統佇列管理程式。
輸出佇列 否 是 OUT 此參數定義目的地佇列。
指定路徑

下列選用名稱空間表格中參數所定義的名稱空間字首及名稱空間 URI，可讓您用來在 XPath 表示式中定義路徑的索引鍵位置。如果包括列，則必須包括這兩個參數。您可以新增、編輯及刪除值，以及變更值在表格中的順序。

型樣參數 M C 預設值 說明
名稱空間字首 是 否 不適用 名稱空間字首參數是併入路徑定義的 XPath 索引鍵位置參數中。
名稱空間 URI 是 否 不適用 名稱空間 URI 是與所指定字首相關聯的名稱空間。

如果遞送參數的值是指定路徑，則下列遞送表中的參數定義輸出目的地及規則。 您可以使用四個參數來定義路徑，而這些參數可以針對需要的路徑數目加以重複。您可以新增、編輯及刪除值，以及變更值在表格中的順序。

型樣參數 M C 預設值 說明
索引鍵位置 是 否 不適用

索引鍵位置是 XPath 表示式，用來定義記錄內正在傳播的位置。例如，$Root/XMLNSC/submitPORequest/partNo。

如需要使用的表示式類型的相關資訊，請參閱 IBM Integration 說明文件中的「遞送節點」與「使用 XPath」。

如果在選用名稱空間表格中指定名稱空間，則可以將其併入。

索引鍵值 是 否 不適用 此參數包含的值是送入訊息中索引鍵位置欄位必須與之相符才能選取此路徑的值。
佇列管理程式 是 是   此參數定義此遞送目的地的佇列管理程式。如果保留為空白，則預設值是分配管理系統佇列管理程式。
佇列 是 是 不適用 此參數定義此遞送目的地的佇列。
查閱路徑
型樣參數 M C 預設值 說明
遞送資料來源 否 是 不適用 此參數定義可從中選取路徑的資料來源。遞送資訊是在 ROUTING_TABLE 表格中進行查閱。
遞送環境定義 否 否 不適用 此參數定義遞送環境定義，讓您可以將 ROUTING_TABLE 用於多個調解流程。

只有 ROUTING_CONTEXT 欄位與遞送資料環境定義值相符的列才會選取來進行遞送查閱，也只有這些列才會新增至記憶體內快取。
索引鍵位置 1 否 否 不適用 此參數定義如何從傳播的訊息元素取出第一個索引鍵。此參數定義為 ESQL 表示式。

第一個索引鍵是用來選取資料來源遞送登錄中的 VARIABLE1。 此位置的格式必須為 InputBody 或 InputRoot。
索引鍵位置 2 否 否 不適用 此參數定義如何從傳播的訊息元素取出選用的第二個索引鍵。此參數定義為 ESQL 欄位參照表示式。

第二個索引鍵是用來選取資料來源遞送登錄中的 VARIABLE2。 此位置的格式必須為 InputBody 或 InputRoot。
快取重新整理 否 否 已清除 如果選取此參數，則在接收到重新整理訊息時，會從資料來源中重新整理遞送資料的記憶體內快取。
重新整理佇列 否 是 REFRESH 如果選取快取重新整理，則傳送至此佇列的訊息會觸發記憶體內遞送資料的重新整理作業。
記載
型樣參數 M C 預設值 說明
需要記載 否 否 已清除 此型樣參數決定型樣實例是否包括用於記載的程式碼。

如果已清除，則不會包括記載子流程。

如果已選取，則必須設定日誌佇列管理程式及日誌佇列的值。

日誌佇列管理程式 否 是   此型樣參數定義佇列管理程式以進行記載。 如果使用分配管理系統佇列管理程式以進行記載，則可以將它保留為空白。

只有在選取需要記載時，才需要它。

日誌佇列 否 是 LOG 此型樣參數定義佇列以進行記載。

只有在選取需要記載時，才需要它。

錯誤處理
型樣參數 M C 預設值 說明
需要錯誤訊息 否 是 已選取 此型樣參數決定型樣實例是否包括用來將錯誤訊息放置於佇列的程式碼。

如果已選取，則必須設定錯誤佇列管理程式及錯誤佇列的值。

錯誤佇列管理程式 否 是   此型樣參數定義佇列管理程式以進行錯誤記載。

只有在選取需要錯誤訊息時，才需要它。

錯誤佇列 否 是 ERROR 此型樣參數定義錯誤訊息的佇列。

只有在選取需要錯誤訊息時，才需要它。

不正確訊息動作 否 否 儲存 此參數定義無法遞送訊息元素時採取的動作。有效值為：
  • 儲存
  • 取消

如果選取儲存，則任何無法遞送的訊息元素都會放置於不正確訊息佇列上。

如果選取取消，而無法遞送訊息元素，則會回復整個訊息流程。

不正確訊息佇列管理程式 否 是   此型樣參數定義佇列管理程式以儲存無法遞送的訊息元素。 如果保留為空白，則預設值是分配管理系統佇列管理程式。

只有在不正確訊息動作參數的值是儲存時，才需要此參數。

不正確訊息佇列 否 是 BAD 此型樣參數定義佇列以儲存無法遞送的訊息元素。

只有在不正確訊息動作參數的值是儲存時，才需要此參數。

型樣參數 M C 預設值 說明
分配管理系統綱目 否 否 mqsi 如果已設定，則會在具名分配管理系統綱目中建立所有訊息流程及 ESQL 檔案。如果未設定，則會使用預設綱目。
流程字首 否 否   如果已設定，則會將此字首新增至所有訊息流程名稱的開頭。
流程字尾 否 否   如果已設定，則會將此字尾新增至所有訊息流程名稱的結尾。
佇列字首 否 否   如果已設定，則會將此字首新增至所有佇列名稱的開頭。

如果使用預設佇列名稱，則必須設定字首及（或）字尾，才能識別每一個型樣實例的佇列名稱。

佇列字尾 否 否   如果已設定，則會將此字尾新增至所有佇列名稱的結尾。
簡要說明 否 否   此說明會新增至所有產生的訊息流程的簡要說明內容。
詳細說明 否 否   此說明會新增至所有產生的訊息流程的詳細說明內容。

所有型樣參數會與其預設值一起列出。

標頭為 M 的直欄指出內容是否為必要的。 標頭為 C 的直欄指出內容是否為可配置的，這表示您可以在將訊息流程新增至分配管理系統保存檔 (BAR) 時變更值。

如果未指出任何預設值，則預設值是空字串。
