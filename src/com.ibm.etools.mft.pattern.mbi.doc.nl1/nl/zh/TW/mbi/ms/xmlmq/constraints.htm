使用 Message Splitter for WebSphere MQ: one-way (for XML) 型樣的限制

當您使用此型樣時，將適用一些限制。

只有在可符合下列所有限制時，才能使用 Message Splitter for WebSphere MQ: one-way (for XML) 型樣：

  • 您的要求者及提供者應用程式使用 WebSphere MQ 作為傳輸機制。
  • 送入訊息是形式完整的 XML；亦即，可剖析但不需要驗證的 XML。
  • 訊息符合預期的結構。每一則輸入訊息都包含：
    • 含括封套，即透過型樣參數選擇性識別的名稱
    • 一系列重複的結構，具有透過型樣參數選擇性識別的名稱
  • 您可以使用下列一種方法來自動處理送入訊息：
    • 所有元素都會寫入輸出或不正確訊息佇列
    • 不會寫入任何元素，並會回復訊息

假設此型樣是用來配送不得遺失的資訊。 因此，此訊息流程會以交易形式執行，並且使用持續訊息，而這對效能有重大影響，但這是必要項目，因為不得遺失資料。

假設重複元素的數目小於輸出佇列上容許的未確定訊息數目上限。如果預期會有大量的重複項目，則可能需要調整此限制。
