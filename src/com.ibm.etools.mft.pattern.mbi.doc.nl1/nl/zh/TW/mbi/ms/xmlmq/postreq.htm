產生 Message Splitter for WebSphere MQ: one-way (for XML) 型樣之後要完成的作業

在您產生此型樣之後必須完成的作業。

  1. 建立分配管理系統保存檔 (BAR)。訊息流程必須新增至分配管理系統保存檔。 會自動新增流程所參照的任何訊息集。
    1. 按一下檔案 > 新建 > BAR 檔。
    2. 輸入您伺服器專案的名稱，或從顯示的清單中進行選取。此清單已經過過濾，只會顯示作用中工作集內的專案。
    3. 輸入所建立 BAR 檔案的名稱。
    4. 按一下完成。 會建立副檔名為 .bar 的檔案，並在「分配管理系統管理導覽器」視圖的「分配管理系統保存檔」資料夾下顯示該檔案。即會開啟 BAR 檔案的「內容」編輯器。
    5. 在「準備」標籤上，選取產生的訊息流程檔案以將其新增至 BAR 檔。
    6. 選取編譯及行內資源，然後按一下建置並儲存以建立 BAR 檔。
  2. 配置分配管理系統保存檔。在您部署 Message Splitter for WebSphere MQ: one-way (for XML) 型樣實例的訊息流程之前，並不需要進一步配置分配管理系統保存檔中的訊息流程，但是您可能會想要檢閱一些預設選項，以及配置環境的變更。

    在「分配管理系統保存檔」編輯器的「管理」頁面中，可執行下列動作：

    1. 選用項目：使用需要記載內容，以選取是要開啟還是關閉訊息分割器訊息流程的記載。
    2. 選用項目：使用需要錯誤訊息內容，以選取是否產生訊息分割器訊息流程的錯誤訊息。
    3. 選用項目：您也可以變更分配管理系統保存檔中的任何可配置內容，例如，安全內容或建立其他實例的內容。您必須確保所變更的任何值不會影響型樣實例的行為。
  3. 建立佇列。當您套用型樣時，會產生具有佇列名稱及佇列管理程式名稱的 WebSphere MQ 佇列清單，如果這些 WebSphere MQ 佇列尚未存在，則必須加以建立。此型樣需要的所有訊息設定都是明確設定於訊息上，而且與佇列設定無關。
    1. 在摘要檔中，檢視您必須建立的佇列清單。摘要檔位於 InstanceName 型樣實例專案的「型樣配置」資料夾中。
    2. 建立 WebSphere MQ 資源。您可以在指令行上使用程式及公用程式（例如 runmqsc），或使用「WebSphere MQ 探險家」圖形介面，以建立 WebSphere MQ 資源。 如需建立 WebSphere MQ 資源的相關資訊，請參閱位於 WebSphere MQ Library 網頁上的 WebSphere MQ Intercommunication。
    3. 如果您在遞送參數中使用查閱路徑值，則可以動態定義路徑，但是路徑不會列在摘要檔中。將路徑新增至資料庫時，必須建立對應的佇列。
  4. 必要的話，設定遞送資料庫。 如果在遞送參數中選取查閱路徑值，則必須使用 ROUTING_TABLE 表格來設定資料來源。此表格必須包括下列直欄：
    • ROUTING_CONTEXT

      使用的值可識別此流程的遞送登錄。

    • VARIABLE1

      訊息元素的 KEY1 內容必須符合的值。

    • VARIABLE2

      訊息元素的 KEY2 內容必須符合的值。

    • QUEUE_MANAGER

      目的地的佇列管理程式。

    • QUEUE

      目的地佇列。
