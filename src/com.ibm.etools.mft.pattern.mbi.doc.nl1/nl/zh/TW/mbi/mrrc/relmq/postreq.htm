產生 Message Correlator for WebSphere MQ: request-response with persistence 型樣之後要完成的作業

在您產生此型樣之後必須完成的作業。

  1. 建立分配管理系統保存檔 (BAR)。將產生的訊息流程新增至 BAR 檔。在「分配管理系統保存檔」編輯器中，您必須先選取編譯及行內資源選項，然後再建置 BAR 檔。 同時必須新增流程所參照的訊息集。
  2. 配置分配管理系統保存檔。在您部署 Message Correlator for WebSphere MQ: request-response with persistence 型樣實例的訊息流程之前，並不需要進一步配置分配管理系統保存檔中的訊息流程，但是您可能會想要檢閱一些預設選項，以及配置環境的變更。
  3. 建立佇列。當您套用型樣時，會產生具有佇列名稱及佇列管理程式名稱的 WebSphere MQ 佇列清單，如果這些 WebSphere MQ 佇列尚未存在，則必須加以建立。此型樣需要的所有訊息設定都是明確設定於訊息上，而且與佇列設定無關。
    1. 在摘要檔中，檢視您必須建立的佇列清單。摘要檔位於 InstanceName 型樣實例專案的「型樣配置」資料夾中。
    2. 建立 WebSphere MQ 資源。您可以在指令行上使用程式及公用程式（例如 runmqsc），或使用「WebSphere MQ 探險家」圖形介面，以建立 WebSphere MQ 資源。 如需建立 WebSphere MQ 資源的相關資訊，請參閱位於 WebSphere MQ Library 網頁上的 WebSphere MQ Intercommunication。
    3. 配置輸入及回應佇列，以作為錯誤導致取消訊息流程時用來保留訊息的取消佇列。
  4. 實作用來監視及更正錯誤的方式。這可能需要監視佇列中的錯誤訊息，或監視事件日誌及取消佇列。
