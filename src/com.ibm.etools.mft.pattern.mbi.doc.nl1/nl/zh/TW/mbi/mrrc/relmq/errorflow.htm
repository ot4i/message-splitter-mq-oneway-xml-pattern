Message Correlator for WebSphere MQ: request-response with persistence 型樣的錯誤子流程

如果在「要求」訊息流程處理期間捕捉到錯誤，則會呼叫「錯誤」子流程。

如果您已將型樣參數配置成包括錯誤訊息，並選取使用者定義的內容 ErrorLoggingOn，則會採取下列動作：

  1. 如果在要求處理期間發生錯誤，則會將錯誤回應傳回給用戶端，指出訊息未順利傳送。

    如果在回應處理期間發生錯誤，則因為回覆位址不明，所以不會傳回錯誤回應。

    下列訊息是標準 XML 錯誤訊息的範例：

    <Error><Status>-1</Status><Text>無法傳送訊息</Text></Error>
  2. 會準備錯誤訊息，並將它寫入錯誤佇列。異常狀況的明細會寫入為 XML 訊息。明細包括：
    • 分配管理系統名稱
    • 流程名稱
    • 時間戳記
    • 異常狀況資料的摘要
  3. 會回復訊息流程，以保留資料。
  4. 錯誤偵測及更正所需的資訊包含在錯誤訊息內。
