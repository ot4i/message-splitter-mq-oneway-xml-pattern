Message Correlator for WebSphere MQ: request-response with persistence 型樣的參數

輸入資訊
型樣參數 M C 預設值 說明
輸入佇列 是 是 IN 此型樣參數識別在其上接收用戶端要求的分配管理系統佇列。
輸入訊息的驗證 否 是 無 此型樣參數識別輸入訊息所需的驗證層次。 有效值為：
  • 無
  • 內容
  • 內容和值
如果驗證設為無，則會將訊息視為不透明 BLOB。

如果驗證指定為內容或內容和值，則需要進一步輸入資訊，用來配置 MQInput 節點。

此內容可以在 BAR 檔案中進行配置。 您可以配置此內容來關閉驗證功能。

除非已定義要求訊息參數，否則您不得使用此內容來開啟驗證功能。
輸入資料類型 否 否 XML 只有在輸入訊息的驗證值是內容或內容和值時，才需要此型樣參數。 有效值為：
  • XML
  • 二進位資料（例如，C 或 COBOL 結構）
  • 文字資料（例如，CSV、SWIFT 或 HL7）
輸入訊息集 否 否  

只有在輸入訊息的驗證值是內容或內容和值時，才需要此型樣參數。

輸入訊息類型 否 否  

只有在輸入訊息的驗證值是內容或內容和值且輸入資料類型值是二進位資料或文字資料時，才需要此型樣參數。
輸入訊息格式 否 否  

只有在輸入訊息的驗證值是內容或內容和值且輸入資料類型值是二進位資料或文字資料時，才需要此型樣參數。
提供者資訊
型樣參數 M C 預設值 說明
提供者要求佇列管理程式 否 是   此型樣參數識別提供者應用程式在其上接收分配管理系統訊息流程中的要求的佇列管理程式。 如果使用分配管理系統佇列管理程式以進行記載，則可以將它保留為空白。
提供者要求佇列 是 是 PROVIDER 此型樣參數識別提供者應用程式在其上接收分配管理系統訊息流程中的要求的佇列。
回應時間上限（以十分之一秒為單位） 是 是 不逾時 此型樣參數定義等待回應的時間上限。

預設值是不逾時；不過，您可以輸入正整數作為相同欄位中逾時期間的值。逾時期間的測量單位是十分之一秒，而且保留所儲存 MQMD 標頭的訊息會與此期限時間一起儲存。

如果值是不逾時，則傳回給分配管理系統的回應一律會傳回給用戶端。

此型樣參數針對保留在佇列中的訊息提供自動內部管理功能，但會避免回應在過了該時段之後與用戶端相符。

儲存佇列 是 是 STORE 此型樣參數定義在等待提供者回應時用來儲存 MQMD 標頭的佇列。

這是分配管理系統的本端佇列。

  • 內容
  • 內容和值

此內容可以在 BAR 檔案中進行配置。 您可以配置此內容來關閉驗證功能。

除非已定義要求訊息參數，否則您不得使用此內容來開啟驗證功能。
回應資料類型 否 否 XML 只有在回應的驗證值是內容或內容和值時，才需要此型樣參數。 有效值為：
  • XML
  • 二進位資料（例如，C 或 COBOL 結構）
  • 文字資料（例如，CSV、SWIFT 或 HL7）
回應訊息集 否 否   只有在回應的驗證值是內容或內容和值時，才需要此型樣參數。

回應訊息格式 否 否   只有在回應的驗證值是內容或內容和值且回應資料類型是二進位資料或文字資料時，才需要此型樣參數。
記載
型樣參數 M C 預設值 說明
需要記載 否 否 已清除

此型樣參數決定型樣實例是否包括用於記載的程式碼。

如果清除需要記載，則不會包括「日誌」子流程。

如果選取需要記載，則「日誌」子流程會併入訊息流程中，而且必須設定日誌佇列管理程式及日誌佇列的值。
日誌佇列管理程式 否 是

此型樣參數定義佇列管理程式以進行記載。 如果使用分配管理系統佇列管理程式以進行記載，則可以將它保留為空白。

只有在選取需要記載時，才需要此參數。
日誌佇列 否 是 LOG

此型樣參數定義佇列以進行記載。

只有在選取需要記載時，才需要此參數。
錯誤處理
型樣參數 M C 預設值 說明
需要錯誤訊息 否 否 已選取

此型樣參數定義型樣應用程式是否建立可建立錯誤訊息的訊息流程元素。

如果選取需要錯誤訊息，則必須設定錯誤佇列管理程式及錯誤佇列的值。
錯誤佇列管理程式 否 是  

此型樣參數定義錯誤訊息的佇列管理程式。 如果使用分配管理系統佇列管理程式以進行記載，則可以將它保留為空白。
錯誤佇列 否 是 ERROR

此型樣參數定義錯誤訊息的佇列。

只有在選取需要錯誤訊息時，才需要此參數。

一般
型樣參數 M C 預設值 說明
分配管理系統綱目 否 否 mqsi 如果已設定，則會在具名分配管理系統綱目中建立所有訊息流程及 ESQL 檔案。如果未設定，則會使用預設綱目。
流程字首 否 否   如果已設定，則會將此字首新增至所有訊息流程名稱的開頭。
流程字尾 否 否   如果已設定，則會將此字尾新增至所有訊息流程名稱的結尾。
佇列字首 否 否   如果已設定，則會將此字首新增至所有佇列名稱的開頭。

如果使用預設佇列名稱，則必須設定字首及（或）字尾，才能識別每一個型樣實例的佇列名稱。
佇列字尾 否 否   如果已設定，則會將此字尾新增至所有佇列名稱的結尾。
簡要說明 否 否   此說明會新增至所有產生的訊息流程的簡要說明內容。
詳細說明 否 否   此說明會新增至所有產生的訊息流程的詳細說明內容。

所有型樣參數會與其預設值一起列出。

標頭為 M 的直欄指出內容是否為必要的。 標頭為 C 的直欄指出內容是否為可配置的，這表示您可以在將訊息流程新增至分配管理系統保存檔 (BAR) 時變更值。

如果未指出任何預設值，則預設值是空字串。
