使用 Message Correlator for WebSphere MQ: request-response without persistence 型樣的限制

當您使用此型樣時，將適用一些限制。

只有在符合下列所有限制時，才能使用 Message Correlator for WebSphere MQ: request-response without persistence 型樣：

  • 您的要求者及提供者應用程式必須使用 WebSphere MQ 作為傳輸機制。
  • 您的提供者應用程式必須遵循將訊息 ID 複製至相關性 ID 的慣例，讓分配管理系統可以符合其原始要求的回應。
  • 除非您在可自訂的 RequestProcessor 子流程中併入轉換功能，否則您的要求者及提供者應用程式必須傳送及接受相容的訊息。
  • 使用此型樣的商業用途必須可以容忍逾時，而且容忍任何後續發生的失敗以接收回覆。
  • 如果您選擇在型樣中啟用記載功能，則會將日誌訊息傳送至佇列。這些日誌訊息的處理超出此型樣的範圍，而且您必須使用最適合組織的方式來處理記載訊息。 選取記載時，會將日誌訊息寫入日誌佇列，以作為持續訊息，而這樣會影響效能。 使用持續訊息的原因是假設已基於審核目的使用記載。如果記載僅供測試或參考之用，則可以配置下列使用者定義的內容 (UDP) RequestLoggingOn 及 ResponseLoggingOn，以在正式作業系統中關閉記載功能。
  • 如果您選擇在型樣中啟用錯誤訊息，則會在發生異常狀況時將錯誤訊息傳送至佇列。監視錯誤以及採取更正動作超出型樣的範圍，而且您必須使用最適合組織的方式來處理錯誤訊息。
  • 此型樣定義它自己的格式以用於記載及錯誤訊息，而且假設這些訊息足以符合您自己環境中的需求。
