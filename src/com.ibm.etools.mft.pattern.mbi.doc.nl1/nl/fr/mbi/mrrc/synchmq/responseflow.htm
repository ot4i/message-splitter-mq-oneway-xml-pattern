Flux Réponse pour le modèle Corrélateur de message pour WebSphere MQ : question-réponse sans persistance

Le flux Réponse est exécuté sous la forme d'un flux non transactionnel.

Lorsqu'une réponse est reçue de l'application fournisseur, les actions suivantes sont exécutées :

  1. Le flux de messages inclut le flux secondaire Processeur de réponses. Ce flux contient uniquement un noeud Passthrough ; par conséquent aucun traitement n'est exécuté. Vous pouvez personnaliser le flux secondaire dans l'instance du modèle pour ajouter votre propre traitement sans changer la structure du modèle.
  2. L'en-tête MQMD enregistré est extrait de la file d'attente de stockage et ajouté au message de réponse pour qu'il dispose de l'adresse appropriée pour répondre au demandeur.
  3. La réponse est envoyée au demandeur.
  4. Le flux de messages peut contenir un flux secondaire pour la consignation, appelé Journal. Vous pouvez configurer le modèle pour inclure ou exclure la consignation. Si la consignation est incluse, le flux secondaire Journal est appelé.
  5. Si la consignation est exclue, le flux de messages se termine correctement et la transaction est validée.
  6. Le flux de messages peut également contenir un flux secondaire, appelé Erreur, pour traiter les erreurs.
  7. Les messages d'erreur sont écrits dans la file d'attente des erreurs si vous avez défini cette option pour l'instance du modèle.
