Flux secondaire Erreur pour le modèle Corrélateur de message pour WebSphere MQ : question-réponse sans persistance

Si des erreurs sont interceptées pendant le traitement du flux de messages de Requête, le flux secondaire Erreur est appelé.

Si vous avez configuré les paramètres du modèle pour inclure des messages d'erreur et que la propriété définie par l'utilisateur ErrorLoggingOn est sélectionnée, les actions suivantes sont effectuées:

  1. Si l'erreur se produit pendant le traitement de la requête, une réponse d'erreur est renvoyée au client, indiquant que le message n'a pas pu être envoyé.

    Si l'erreur se produit pendant le traitement de la réponse, aucune réponse d'erreur n'est renvoyée car l'adresse de réponse est inconnue.

    Le message suivant est un exemple de message d'erreur XML standard :

    <Erreur><Etat>-1</Etat><Texte>Echec de l'envoi du message</Texte></Erreur>
  2. Un message d'erreur est écrit dans la file d'attente des erreurs. Les détails de l'exception sont écrits sous forme de message XML. Les détails comprennent :
    • Nom du courtier
    • Nom du flux
    • Horodatage
    • Récapitulatif des données de l'exception
