Paramètres du modèle Corrélateur de message pour WebSphere MQ : demande-réponse sans persistance

Informations d'entrée
Paramètres du modèle O C Valeur par défaut Description
File d'entrée Oui Oui IN Ce paramètre du modèle définit la file d'attente du courtier dans laquelle sont placées les demandes des clients.
Validation des messages entrants Non Oui Aucun Ce paramètre du modèle définit le niveau de validation requis pour les messages entrants. Valeurs acceptées :
  • Aucun
  • Contenu
  • Contenu et valeur
Si la validation est affectée de la valeur Aucun, le message est traité comme objet BLOB opaque.

Si la validation est spécifiée en tant que Contenu ou Contenu et valeur, d'autres informations en entrée sont requises, qui sont utilisées pour configurer le noeud MQInput node.

Cette propriété peut être configurée dans le fichier BAR. Vous pouvez configurer cette propriété pour désactiver la validation.

Vous ne devez pas utiliser cette propriété pour activer la validation, sauf si des paramètres de message de demande ont été définis.
Type de données en entrée Non Non XML Ce paramètre du modèle n'est requis que si la valeur de Validation des messages entrants est Contenu ou Contenu et valeur. Valeurs acceptées :
  • XML
  • Données binaires (par exemple, structure C ou COBOL)
  • Données texte (CSV, SWIFT, ou HL7 par exemple)
Ensemble de messages entrants Non Non   Ce paramètre du modèle n'est requis que si la valeur de Validation des messages entrants est Contenu ou Contenu et valeur.
Type de message entrant Non Non   Ce paramètre du modèle n'est requis que si la valeur de Validation des messages entrants est Contenu ou Contenu et valeur, et que la valeur de Type de données de la demande est Données binaires ou Données de texte.
Format de message entrant Non Non   Ce paramètre du modèle n'est requis que si la valeur de Validation des messages entrants est Contenu ou Contenu et valeur, et que la valeur de Type de données d'entrée est Données binaires ou Données de texte.
Informations sur le fournisseur
Paramètre du modèle O C Valeur par défaut Description
Gestionnaire de file d'attente des demandes du fournisseur Non Oui

Ce paramètre définit le gestionnaire de files d'attente qui reçoit toutes les demandes destinées au fournisseur.
File d'attente des demandes du fournisseur Oui Oui PROVIDER

Ce paramètre définit la file d'attente qui reçoit les messages de demande du fournisseur.
Délai de réponse maximal en dixièmes de seconde Oui Oui Pas de délai d'attente

Ce paramètre du modèle définit le délai d'attente maximum d'une réponse.

La valeur par défaut est Aucun délai d'attente. Toutefois, vous pouvez entrer un entier positif comme valeur de délai d'attente dans la même zone. Le délai d'attente est mesuré en dixièmes de seconde et les messages qui contiennent des en-têtes MQMD enregistrés sont enregistrés avec ce délai d'expiration.

Si la valeur est Aucun délai d'attente, les réponses du courtier sont toujours envoyées au client.

Ce paramètre du modèle nettoie automatiquement les messages qui restent dans la file d'attente, mais empêche d'associer les réponses à un client après l'expiration du délai.

File d'attente de stockage Oui Oui STORE

Ce paramètre du modèle définit la file d'attente utilisée pour stocker les en-têtes MQMD lors de l'attente d'une réponse du fournisseur.

Cette file d'attente est locale pour le courtier.
Informations de réponse
Paramètre du modèle O C Valeur par défaut Description
File d'attente de réponses Oui Oui RESPONSE Ce paramètre du modèle identifie la file d'attente du courtier dans laquelle sont placées les réponses du fournisseur.
Validation des messages de réponse Non Oui Aucun Ce paramètre du modèle définit le niveau de validation requis des messages de réponse. Les valeurs valides sont :
  • Aucun
  • Contenu
  • Contenu et valeur

Cette propriété peut être configurée dans le fichier BAR. Vous pouvez configurer cette propriété pour désactiver la validation.

Vous ne devez pas utiliser cette propriété pour activer la validation, sauf si des paramètres de message de demande ont été définis.
Type de données de réponse Non Non XML

Ce paramètre du modèle n'est requis que si la valeur de Validation de la réponse est Contenu ou Contenu et valeur.

Valeurs acceptées :
  • XML
  • Données binaires (par exemple, structure C ou COBOL)
  • Données texte (CSV, SWIFT, ou HL7 par exemple)
Ensemble de messages de réponse Non Non   Ce paramètre du modèle n'est requis que si la valeur de Validation de la réponse est Contenu ou Contenu et valeur.
Type de message de réponse Non Non   Ce paramètre du modèle n'est obligatoire que si la valeur de Validation de la réponse est Contenu ou Contenu et valeur, et que la valeur de Type de données de réponse est Données binaires ou Données texte.
Format de message de réponse Non Non   Ce paramètre du modèle n'est obligatoire que si la valeur de Validation de la réponse est Contenu ou Contenu et valeur, et que la valeur de Type de données de réponse est Données binaires ou Données texte.
Consignation
Paramètre du modèle O C Valeur par défaut Description
Consignation nécessaire Non Non Effacé

Ce paramètre du modèle indique si l'instance du modèle contient le code de consignation.

Si Consignation nécessaire est désélectionné, aucun flux secondaire Journal n'est inclus.

Si Consignation nécessaire est sélectionné, un flux secondaire Journal est inclus dans le flux de messages et vous devez définir les valeurs de Gestionnaire de files d'attente de consignation et File d'attente de journaux.
Gestionnaire de files d'attente de journaux Non Oui

Ce paramètre du modèle définit le gestionnaire de files d'attente de la consignation. Vous pouvez ne pas le définir si vous utilisez le gestionnaire de files d'attente du courtier pour la consignation.

Vous devez le définir uniquement si vous sélectionnez Consignation nécessaire.

File d'attente de consignation Non Oui LOG

Ce paramètre définit la file d'attente de consignation.

Vous devez le définir uniquement si vous sélectionnez Consignation nécessaire.
Traitement d'erreurs
Paramètre du modèle O C Valeur par défaut Description
Message d'erreur nécessaire Non Non Sélectionné

Ce paramètre de modèle indique si l'application de modèle crée les éléments de flux de messages pour créer les messages d'erreur.

Si vous sélectionnez Message d'erreur nécessaire, vous devez définir les valeurs de Gestionnaire de file d'attente d'erreurs et File d'attente d'erreurs.
Gestionnaire de files d'attente d'erreurs Non Oui  

Ce paramètre de modèle définit le gestionnaire de files d'attente des messages d'erreur. Vous pouvez ne pas le définir si vous utilisez le gestionnaire de files d'attente du courtier pour la consignation.
File d'attente d'erreurs Non Oui ERROR

Ce paramètre du modèle définit la file d'attente des messages d'erreur.

Vous devez le définir uniquement si vous sélectionnez Message d'erreur nécessaire.

Dispositions générales
Paramètre du modèle O C Valeur par défaut Description
Schéma de courtier Non Non mqsi

Si un schéma de courtier est défini, tous les flux de messages et fichiers ESQL sont créés dans le schéma. Si aucun schéma n'est défini, le schéma de courtier par défaut est utilisé.
Préfixe de flux Non Non

Si vous définissez ce préfixe, il est ajouté au début de tous les noms de flux de messages.
Suffixe de flux Non Non

Si vous définissez ce suffixe, il est ajouté à la fin de tous les noms de flux de messages.
Préfixe de file d'attente Non Non

Si vous définissez ce préfixe, il est ajouté au début de tous les noms de file d'attente.

Si vous utilisez les noms de file d'attente par défaut, vous devez définir le préfixe, le suffixe ou les deux pour identifier les noms de file d'attente de chaque instance du modèle.

Suffixe de file d'attente Non Non

Si vous définissez ce suffixe, il est ajouté à la fin de tous les noms de file d'attente.

Si vous utilisez les noms de file d'attente par défaut, vous devez définir le préfixe, le suffixe ou les deux pour identifier les noms de file d'attente de chaque instance du modèle.

Description abrégée Non Non

Cette description est ajoutée à la propriété Description abrégée de tous les flux de messages générés.
Description détaillée Non Non

Cette description est ajoutée à la propriété Description détaillée de tous les flux de messages générés.

Tous les paramètres du modèle sont répertoriés avec leur valeur par défaut.

La colonne avec l'en-tête M indique si la propriété est obligatoire. La colonne avec l'en-tête C indique si la propriété est configurable, ce qui signifie que vous pouvez modifier la valeur lorsque vous ajoutez le flux de messages au fichier d'archive du courtier (BAR).

Lorsqu'aucune valeur n'est définie, la valeur par défaut est une chaîne vide.
