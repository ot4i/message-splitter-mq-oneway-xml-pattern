Flux secondaire Journal pour le modèle Corrélateur de message pour WebSphere MQ : question-réponse avec persistance

Si le flux secondaire Journal est inclus dans le flux Demande, la consignation des demandes est contrôlée par la propriété RequestLoggingOn définie par l'utilisateur. Si le flux secondaire Journal est inclus dans le flux Réponse, la consignation des réponses est contrôlée par la propriété ResponseLoggingOn définie par l'utilisateur.

Si vous sélectionnez la propriété RequestLoggingOn, la consignation est requise, le flux secondaire de consignation est appelé et termine le flux. Si la consignation échoue, l'exception est interceptée dans le flux et les informations sont écrites dans un noeud Trace.

Si la consignation est incluse dans le flux Réponse, le flux secondaire Journal est appelé. Toutes les erreurs de consignation sont interceptées ; le flux se termine correctement et la transaction du flux est validée.

Les messages de journal sont écrits sous forme de messages persistants WebSphere MQ dans une file d'attente WebSphere MQ. Ces messages de journal sont ensuite traités séparément conformément aux pratiques standard de votre entreprise.

Le journal de messages contient le message d'origine et les informations suivantes dans un en-tête MQRFH2 :
  • Horodatage
  • Nom du courtier
  • Nom du flux de messages
