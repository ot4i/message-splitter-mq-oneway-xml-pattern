Paramètres du modèle d'utilitaire de Fractionnement de message pour WebSphere MQ : unidirectionnel (pour XML)

Informations d'entrée
Paramètre du modèle O C Valeur par défaut Description
File d'entrée Oui Oui IN Ce paramètre du modèle définit la file d'attente du courtier dans laquelle sont placés les messages XML.
Nom de conteneur Oui Non *:* Chemin de l'élément conteneur qui se trouve dans le message XML entrant qui contient les éléments à extraire. Ce chemin est défini avec une syntaxe de type ESQL, mais il inclut uniquement les éléments du message XML, tels que SalesData.Invoices.

L'espace de nom peut être fourni en entier, et s'il n'est pas nécessaire d'établir de distinction entre deux éléments conteneur de même nom ayant des espaces de nom différents, les espaces de nom peuvent être fournis sous une forme générique, par exemple *:SalesData.*:Invoices.

Si vous utilisez la valeur par défaut *:*, l'élément racine du message XML est utilisé par défaut et il peut avoir n'importe quel nom.

Nom d'élément de message Oui Non *:* Nom des éléments extraits du message entrant.

Si vous utilisez la valeur par défaut, *:*, les enfants du conteneur sont acceptés.

Les espaces de nom ne doivent pas être définis, car tout élément portant le nom fourni est supposé nécessaire, quel que soit l'espace de nom.

File d'attente de sortie Non Oui OUT Ce paramètre du modèle définit la file d'attente de destination.
Indiquer les routes

Les paramètres dans la table Espace de nom facultative suivante, définissent les préfixes et les URI d'espace de nom que vous pouvez utiliser pour définir les emplacements clés des routes dans des expressions XPath. Si une ligne est incluse, les deux paramètres doivent être inclus. Vous pouvez ajouter, modifier et supprimer les valeurs et changer leur ordre dans la table.

Paramètre du modèle O C Valeur par défaut Description
Préfixe d'espace de nom Oui Non Non disponible Le paramètre Préfixe d'espace de nom est inclus dans le paramètre Emplacement de clé XPath dans la définition des routes.
URI d'espace de nom Oui Non Non disponible L'URI d'espace de nom correspond à l'espace de nom associé au préfixe spécifié.

Les paramètres dans la table de routage définissent les destinations et les règles de sortie si la valeur du paramètre Routage est Indiquer les routes. Quatre paramètres sont utilisés pour définir une route ; ils peuvent être répétés pour autant de routes que nécessaire. Vous pouvez ajouter, modifier et supprimer les valeurs et changer leur ordre dans la table.

Paramètre du modèle O C Valeur par défaut Description
Emplacement de clé Oui Non Non disponible

Un emplacement de clé est une expression XPath qui définit un emplacement dans l'enregistrement propagé. Par exemple, $Root/XMLNSC/submitPORequest/partNo.

Pour plus d'informations sur le type d'expression à utiliser, voir "Noeud de route" et "Utilisation de XPath" dans la documentation IBM Integration.

Vous pouvez inclure des espaces de nom s'ils sont définis dans la table Espace de nom facultative.

Valeur de clé Oui Non Non disponible Ce paramètre contient la valeur à laquelle doit correspondre la zone Emplacement de clé dans le message entrant pour sélectionner la route.
Gestionnaire de file d'attente Oui Oui   Ce paramètre définit le gestionnaire de files d'attente de la destination du routage. Si vous ne le définissez pas, le gestionnaire de files d'attente du courtier est utilisé par défaut.
File d'attente Oui Oui Non disponible Ce paramètre définit la file d'attente de la destination du routage.
Rechercher les routes
Paramètre du modèle O C Valeur par défaut Description
Source des données de routage Non Oui Non disponible Ce paramètre définit la source de données dans laquelle une route peut être sélectionnée. Les informations de routage sont recherchées dans la table ROUTING_TABLE.
Contexte de routage Non Non Non disponible Ce paramètre définit un contexte de routage pour que vous puissiez utiliser une table ROUTING_TABLE pour plusieurs flux de médiation.

Seules les lignes dans lesquelles la zone ROUTING_CONTEXT correspond à la valeur Contexte de routage sont sélectionnées pour la recherche de routes et ajoutées à la mémoire cache interne.
Emplacement de clé 1 Non Non Non disponible Ce paramètre indique le mode d'extraction de la première clé de l'élément de message propagé. Il est défini sous la forme d'une expression ESQL.

La première clé est utilisée pour sélectionner VARIABLE1 dans une entrée de routage. L'emplacement doit être sous la forme InputBody ou InputRoot.
Emplacement de clé 2 Non Non Non disponible Ce paramètre définit le mode d'extraction de la seconde clé de l'élément de message propagé. Il est défini sous la forme d'une expression de référence de zone ESQL.

La deuxième clé est utilisée pour sélectionner VARIABLE2 dans une entrée de routage source. L'emplacement doit être sous la forme InputBody ou InputRoot.
Régénération du cache Non Non Effacé Si vous sélectionnez ce paramètre, la mémoire cache interne des données de routage est régénérée depuis la source de données lorsqu'un message de régénération est reçu.
Régénérer la file d'attente Non Oui REFRESH Si vous sélectionnez Régénérer le cache, un message est envoyé à cette file d'attente qui déclenche une régénération des données de routage dans la mémoire interne.
Consignation
Paramètre du modèle O C Valeur par défaut Description
Consignation nécessaire Non Non Effacé Ce paramètre du modèle indique si l'instance du modèle contient le code de consignation.

Si vous sélectionnez Effacé, le flux secondaire de consignation n'est pas inclus.

Si vous le sélectionnez, vous devez définir les valeurs Gestionnaire de files d'attente de journaux et file d'attente de journaux.

Gestionnaire de files d'attente de journaux Non Oui   Ce paramètre du modèle définit le gestionnaire de files d'attente de consignation Vous pouvez ne pas le définir si le gestionnaire de files d'attente du courtier est utilisé pour la consignation.

Vous ne devez le définir que si vous sélectionnez Consignation nécessaire.

File d'attente de journaux Non Oui JOURNAL Ce paramètre du modèle définit la file d'attente de consignation.

Vous ne devez le définir que si vous sélectionnez Consignation nécessaire.

Traitement d'erreurs
Paramètre du modèle O C Valeur par défaut Description
Message d'erreur nécessaire Non Oui Sélectionné Ce paramètre indique si l'instance du modèle contient le code d'insertion des messages d'erreur dans une file d'attente.

Si vous le sélectionnez, vous devez définir les valeurs de Gestionnaire de files d'attente d'erreurs et File d'attente d'erreurs.

Gestionnaire de files d'attente d'erreurs Non Oui   Ce paramètre du modèle définit le gestionnaire de files d'attente de consignation des erreurs.

Vous ne devez le définir que si Message d'erreur nécessaire est sélectionné.

File d'attente d'erreurs Non Oui ERROR Ce paramètre du modèle définit la file d'attente des messages d'erreur.

Vous ne devez le définir que si Message d'erreur nécessaire est sélectionné.

Action de message incorrect Non Non Enregistrer Ce paramètre définit l'action à exécuter lorsqu'un élément de message ne peut pas être routé. Les valeurs valides sont :
  • Enregistrer
  • Annuler

Si vous sélectionnez Enregistrer, tous les éléments de message qui ne peuvent pas être routés sont placés dans la file d'attente des messages incorrects.

Si vous sélectionnez Annuler et qu'un élément de message ne peut pas être routé, l'ensemble du flux de messages est annulé.

Gestionnaire de file d'attente des messages incorrects Non Oui   Ce paramètre du modèle définit le gestionnaire de files d'attente d'enregistrement des éléments de message qui ne peuvent pas être routés. Si vous ne le définissez pas, le gestionnaire de files d'attente du courtier est utilisé.

Vous devez définir ce paramètre uniquement si la valeur du paramètre Action de message incorrect est Enregistrer.

File d'attente des messages incorrects Non Oui BAD Ce paramètre définit la file d'attente d'enregistrement des éléments de message qui ne peuvent pas être routés.

Vous devez définir ce paramètre uniquement si la valeur du paramètre Action de message incorrect est Enregistrer.

Dispositions générales
Paramètre du modèle O C Valeur par défaut Description
Schéma de courtier Non Non mqsi Si vous définissez ce paramètre, tous les flux de messages et les fichiers ESQL sont créés dans le schéma de courtier nommé. Si vous ne le définissez pas, le schéma par défaut est utilisé.
Préfixe de flux Non Non   Si vous définissez ce préfixe, il est ajouté au début de tous les noms de flux de messages.
Suffixe de flux Non Non   Si vous définissez ce suffixe, il est ajouté à la fin de tous les noms de flux de messages.
Préfixe de la file d'attente Non Non   Si vous définissez ce préfixe, il est ajouté au début de tous les noms de file d'attente.

Si vous utilisez les noms de file d'attente par défaut, vous devez définir le préfixe, le suffixe ou les deux pour identifier les noms de file d'attente de chaque instance du modèle.

Suffixe de la file d'attente Non Non   Si vous définissez ce suffixe, il est ajouté à la fin de tous les noms de file d'attente.
Description abrégée Non Non   Cette description est ajoutée à la propriété Description abrégée de tous les flux de messages générés.
Description détaillée Non Non   Cette description est ajoutée à la propriété Description détaillée de tous les flux de messages générés.

Tous les paramètres du modèle sont répertoriés avec leurs valeurs par défaut.

La colonne avec l'en-tête M indique si la propriété est obligatoire. La colonne avec l'en-tête C indique si la propriété est configurable, ce qui signifie que vous pouvez modifier la valeur lorsque vous ajoutez le flux de messages au fichier d'archive du courtier (BAR).

Lorsqu'aucune valeur n'est définie, la valeur par défaut est une chaîne vide.
