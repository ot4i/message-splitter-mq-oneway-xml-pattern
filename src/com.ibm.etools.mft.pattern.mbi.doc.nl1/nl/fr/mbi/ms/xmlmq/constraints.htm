Contraintes sur l'utilisation du modèle d'utilitaire de Fractionnement de message pour WebSphere MQ : unidirectionnel (pour XML)

Il existe certaines contraintes lorsque vous utilisez le modèle.

Vous ne pouvez utiliser le modèle Utilitaire de fractionnement de message Message WebSphere MQ : unidirectionnel (pour XML) que si toutes les contraintes suivantes sont respectées :

  • Vos applications demandeur et fournisseur doivent utiliser WebSphere MQ comme mécanisme de transport.
  • Les messages entrants doivent avoir le format XML adéquat, à savoir que le code XML analyse, mais ne valide pas nécessairement.
  • Les messages doivent être conformes à la structure attendue. Chaque message d'entrée contient :
    • Une enveloppe fermée dont le nom est identifié facultativement par un paramètre du modèle.
    • Une série de structures répétées, les noms étant identifiés facultativement par un paramètre du modèle.
  • Le message entrant doit être traité de manière atomique d'une des deux manières suivantes :
    • Tous les éléments sont écrits sur la file d'attente de sortie ou des messages erronés
    • Aucun élément n'est écrit et le message est annulé

Ce modèle est conçu pour distribuer de informations qui ne doivent pas être perdues. Par conséquent, le flux de messages est exécuté sous la forme d'une transaction et utilise des messages persistants, ce qui a un impact significatif sur les performances, mais est nécessaire, car aucune donnée ne doit être perdue.

Le nombre d'éléments répétés doit être inférieur au nombre maximum de messages non validés autorisés dans la file d'attente de sortie. Il peut être nécessaire d'ajuster cette limite si de nombreuses répétitions sont prévues.
