Tâches à exécuter avant d'appliquer le modèle d'utilitaire de Fractionnement de message pour WebSphere MQ : unidirectionnel (pour XML)

Avant d'appliquer le modèle pour produire une instance du modèle, vous devez déterminer les tâches à exécuter parmi les tâches suivantes. Si nécessaire, exécutez les tâches suivantes :

  • Sélectionnez un nom unique pour l'instance du modèle. Vous pouvez utiliser le modèle plusieurs fois et créer plusieurs instances différentes, qui se distinguent par leur nom unique. Identifiez l'objectif de l'application de modèle, afin de déterminer un nom unique approprié ou significatif.
  • Sélectionnez les gestionnaires de files d'attente et les files d'attente WebSphere MQ que doit utiliser l'instance du modèle, par exemple, pour recevoir le message XML. Si vous utilisez des files d'attente existantes, vous devez les enregistrer. Si vous voulez créer des gestionnaires de files d'attente et des files d'attente, vous devez définir le nom de ces ressources.
  • Indiquez si la validation est nécessaire et définissez un ensemble de messages ou définissez le nom de l'ensemble de messages. Lorsque le modèle a été généré, terminez la tâche de création de l'ensemble de messages.
  • Identifiez les gestionnaires de files d'attente et les files d'attente supplémentaires nécessaires aux modèles et définissez les noms à utiliser. Tenez compte des règles, des normes ou des conventions d'attribution de nom qui s'appliquent.
  • Déterminez s'il est nécessaire de consigner les messages entrants. Si vous devez les consigner, vous devez définir le gestionnaire de files d'attente et la file d'attente auxquels les messages de consignation sont envoyés. Vous pouvez envoyer les messages de consignation vers une file d'attente de consignation générale utilisée par plusieurs flux de messages ou utiliser une file d'attente spécifique pour l'instance du modèle.
  • Indiquez s'il existe plusieurs destinations possibles. Si tel est le cas, sélectionnez le mode de routage à utiliser.
    • Si vous disposez d'un nombre limité de routes dans lesquelles les destinations ne changent pas souvent, affectez au paramètre Routage la valeur Indiquer les routes.
    • Si le groupe de routes cible n'est pas complètement déterminé ou peut changer fréquemment, affectez au paramètre Routage la valeur Rechercher les routes. Si vous sélectionnez cette valeur, le nom de la source de données doit être connu.
  • Définissez le mode de gestion des erreurs. Indiquez si les éléments de message qui ne peuvent pas être routés correctement doivent déclencher une annulation complète ou qu'il suffit d'écrire l'élément ayant échoué dans une file d'attente pour traiter l'exception.
  • Indiquez si vous voulez générer des messages d'erreur pour l'instance du modèle. Toutes les données sont conservées sans messages d'erreur, car les actions d'erreur laissent les éléments dans une file d'attente à cet effet ou annulent le flux de messages. le message entrant est soit renvoyé à la file d'attente entrante, soit envoyé dans une file d'attente d'annulation. Toutefois, si ces files d'attente ne sont pas contrôlées, des messages d'erreur peuvent être nécessaires comme notifications.

    Si décidez d'envoyer les messages d'erreur dans l'instance du modèle, vous pouvez activer ou désactiver cette fonction en configurant le fichier archive de courtier (BAR) avant le déploiement.
