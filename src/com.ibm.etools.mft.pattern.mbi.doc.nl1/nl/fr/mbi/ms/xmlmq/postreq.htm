Tâches à exécuter après avoir généré le modèle d'utilitaire de Fractionnement de message pour WebSphere MQ : unidirectionnel (pour XML)

Tâches que vous devez exécuter après avoir généré le modèle.

  1. Créez le fichier archive de courtier (BAR). Le flux de messages doit être ajouté à un fichier archive de courtier. Les ensembles de messages référencés par le flux sont ajoutés automatiquement.
    1. Cliquez sur Fichier > Nouveau > Fichier BAR.
    2. Entrez le nom du projet de serveur ou sélectionnez-en un dans la liste affichée. La liste est filtrée pour contenir uniquement les projets dans l'ensemble actif de projets.
    3. Entrez le nom du fichier BAR à créer.
    4. Cliquez sur Terminer. Un fichier portant l'extension .bar est créé et s'affiche dans la vue Explorateur d'administration du courtier sous le dossier Archives de courtier. L'éditeur de contenu du fichier BAR s'affiche.
    5. Dans l'onglet Préparation, sélectionnez le fichier de flux de messages généré et ajoutez-le au fichier BAR.
    6. Sélectionnez Compiler et intégrer des ressources et cliquez ensuite sur Générer et sauvegarder pour créer le fichier BAR.
  2. Configurez le fichier archive du courtier. Il n'est pas nécessaire d'effectuer d'autres opérations de configuration au niveau du flux de messages dans le fichier archive de courtier avant de déployer les flux de messages de votre Utilitaire de fractionnement de message WebSphere MQ. Néanmoins, vous pouvez vérifier certaines options par défaut et configurer des modifications pour votre environnement.

    Dans la page Gestion de l'éditeur d'archive de courtier :

    1. Facultatif : utilisez la propriété Consignation nécessaire pour indiquer si la consignation est activée ou désactivée pour le flux de messages de l'utilitaire de fractionnement de message.
    2. Facultatif : utilisez la propriété Message d'erreur nécessaire pour indiquer si des messages d'erreur sont générés pour le flux de messages de l'utilitaire de fractionnement de message.
    3. Facultatif : vous pouvez également modifier les propriétés configurables dans l'archive de courtier, telles que les propriétés de sécurité ou de création d'instances supplémentaires. Vous devez vérifier que les valeurs que vous changez n'affectent pas le comportement de l'instance de modèle.
  3. Créez les files d'attente. Lorsque vous appliquez le modèle, la liste des files d'attente WebSphere MQ est générée, avec les noms de file d'attente et de gestionnaire de file d'attente, que v vous devez créer s'ils n'existent pas. Tous les paramètres de message nécessaires au modèle sont définis explicitement dans les messages et ne dépendent pas des paramètres de file d'attente.
    1. Dans le fichier récapitulatif, consultez la liste des files d'attente que vous devez créer. Ce fichier se trouve dans le dossier Configuration du modèle du projet d'instance de modèle InstanceName.
    2. Créez les ressources WebSphere MQ. Vous pouvez créer les ressources WebSphere MQ depuis la ligne de commande, en utilisant des programmes et des utilitaires, tels que runmqsc, ou en utilisant l'interface graphique WebSphere MQ Explorer. Pour plus d'informations sur la création des ressources WebSphere MQ, voir WebSphere MQ Intercommunication, sur la page Web Bibliothèque WebSphere MQ.
    3. Si vous utilisez la valeur Rechercher les routes dans le paramètre Routage, vous pouvez définir les routes dynamiquement dans la base de données, mais ces dernières ne figurent pas dans le fichier récapitulatif. Lorsque vous ajoutez des routes à la base de données, vous devez créer les files d'attente correspondantes.
  4. Configurez une base de données de routage, si nécessaire. Si la valeur Rechercher les routes est sélectionnée dans le paramètre Routage, vous devez configurer la source de données avec une ROUTING_TABLE. Cette table doit contenir les colonnes suivantes :
    • ROUTING_CONTEXT

      Utilisez une valeur qui identifie les entrées de routage du flux.

    • VARIABLE1

      Valeur à laquelle la propriété KEY1 de l'élément de message doit correspondre.

    • VARIABLE2

      Valeur à laquelle la propriété KEY2 de l'élément de message doit correspondre.

    • QUEUE_MANAGER

      Gestionnaire de files d'attente de destination.

    • QUEUE

      File d'attente de destination.
