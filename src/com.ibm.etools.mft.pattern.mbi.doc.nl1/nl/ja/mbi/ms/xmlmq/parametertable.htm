パターン・パラメーター M C デフォルト 説明
入力キュー はい はい IN このパターン・パラメーターは、XML メッセージを受け取るブローカー・キューを識別します。
コンテナー名 はい いいえ *:* 抽出するエレメントを保持する、着信 XML メッセージに含まれるコンテナー・エレメントのパス。 このパスは ESQL のような構文で指定されますが、XML のエレメント (例えば、SalesData.Invoices) のみを含みます。

ネーム・スペースは完全に指定できますが、名前が同じでネーム・スペースが異なる 2 つのコンテナー・エレメントを区別する必要がない限り、ネーム・スペースをワイルドカードとして指定することができます。例: *:SalesData.*:Invoices。

デフォルト値 *:* が使用されている場合、XML メッセージのルート・エレメントと見なされ、名前は任意のものでかまいません。

メッセージ・エレメント名 はい いいえ *:* 着信メッセージから抽出されるエレメントの名前。

デフォルト値 *:* が使用されている場合、コンテナーのどの子も受け入れられます。

ネーム・スペースを指定すると、ネーム・スペースに関係なく、指定された名前の任意のエレメントが必要であると見なされるため、指定しないでください。

経路の指定

以下のオプションの「ネーム・スペース・テーブル」のパラメーターは、XPath 式で経路のキーの位置を定義するために使用できる、ネーム・スペース接頭部およびネーム・スペース URI を定義します。 行が含まれる場合、両方のパラメーターを含める必要があります。 テーブルでは、値を追加、編集、および削除したり、値の順序を変更したりすることができます。

パターン・パラメーター M C デフォルト 説明
ネーム・スペース接頭部 はい いいえ 適用外 経路の定義では、「ネーム・スペース接頭部」パラメーターは、XPath「キーの位置」パラメーターに含まれます。
ネーム・スペース URI はい いいえ 適用外 「ネーム・スペース URI」は、指定された接頭部に関連付けられるネーム・スペースです。

「ルーティング」パラメーターの値が「経路の指定」である場合、以下の「ルーティング・テーブル」のパラメーターは、出力宛先および規則を定義します。 4 つのパラメーターは、経路を定義するために使用でき、必要な数の経路に対して繰り返すことができます。 テーブルでは、値を追加、編集、および削除したり、値の順序を変更したりすることができます。

パターン・パラメーター M C デフォルト 説明
キーの位置 はい いいえ 適用外

キーの位置は、伝搬されるレコード内の位置を定義する XPath 式です。例えば、$Root/XMLNSC/submitPORequest/partNo というものです。

使用する式のタイプについて詳しくは、IBM Integration 資料の『Route ノード』および『XPath の使用』の項を参照してください。

オプションの「ネームスペース・テーブル」で指定されている場合には、ネーム・スペースを含めることができます。

キー値 はい いいえ 適用外 このパラメーターには、この経路を選択するために着信メッセージの「キー・ロケーション」フィールドに一致する必要のある値が含まれます。
キュー・マネージャー はい はい   このパラメーターは、このルーティング宛先のキュー・マネージャーを定義します。 ブランクのままにすると、デフォルトはブローカー・キュー・マネージャーです。
キュー はい はい 適用外 このパラメーターは、このルーティング宛先のキューを定義します。
経路の検索
パターン・パラメーター M C デフォルト 説明
ルーティング・データ・ソース いいえ はい 適用外 このパラメーターは、経路の選択元にできるデータ・ソースを定義します。 ルーティング情報は ROUTING_TABLE テーブルで検索されます。
ルーティング・コンテキスト いいえ いいえ 適用外 このパラメーターは、複数のメディエーション・フローで ROUTING_TABLE を使用できるように、ルーティング・コンテキストを定義します。

ルーティング検索で選択され、メモリー内のキャッシュに追加されるのは、ROUTING_CONTEXT フィールドが「ルーティング・データ・コンテキスト」の値と一致する行のみです。

キーの位置 1 いいえ いいえ 適用外 このパラメーターは、伝搬されるメッセージ・エレメントから最初のキーを抽出する方法を定義します。 このパラメーターは ESQL 式として定義されます。

最初のキーは、データ・ソース・ルーティング項目の VARIABLE1 での選択に使用されます。場所は、InputBody または InputRoot の形式でなければなりません。
キーの位置 2 いいえ いいえ 適用外 このパラメーターは、オプションの 2 番目のキーを伝搬メッセージ・エレメントから抽出する方法を定義します。 このパラメーターは ESQL フィールド参照式として定義されます。

2 番目のキーは、データ・ソース・ルーティング項目の VARIABLE2 での選択に使用されます。場所は、InputBody または InputRoot の形式でなければなりません。
キャッシュ最新表示 いいえ いいえ クリア済み このパラメーターを選択した場合、ルーティング・データのメモリー内キャッシュは、最新表示メッセージを受け取ったときにデータ・ソースから最新表示されます。
キューの最新表示 いいえ はい REFRESH 「キャッシュ最新表示」が選択されている場合、このキューに送られたメッセージは、メモリー内のルーティング・データの最新表示を起動します。
ロギング
パターン・パラメーター M C デフォルト 説明
必要なロギング いいえ いいえ クリア済み このパターン・パラメーターは、ロギング用のコードをパターン・インスタンスに含めるかどうかを決定します。

クリアされている場合、ロギング・サブフローは含められません。

選択されている場合、「ログ・キュー・マネージャー」および「ログ・キュー」値を設定する必要があります。

ログ・キュー・マネージャー いいえ はい   このパターン・パラメーターは、ロギング用のキュー・マネージャーを定義します。 ロギングにブローカー・キュー・マネージャーが使用される場合、これをブランクのままにすることができます。

これは、「必要なロギング」が選択されている場合のみ必要です。

ログ・キュー いいえ はい LOG このパターン・パラメーターは、ロギング用のキューを定義します。

これは、「必要なロギング」が選択されている場合のみ必要です。

エラー処理
パターン・パラメーター M C デフォルト 説明
必要なエラー・メッセージ いいえ はい 選択済み このパターン・パラメーターは、キューにエラー・メッセージを入れるためのコードをパターン・インスタンスに含めるかどうかを決定します。

選択されている場合、「エラー・キュー・マネージャー」および「エラー・キュー」値を設定する必要があります。

エラー・キュー・マネージャー いいえ はい   このパターン・パラメーターは、エラー・ロギング用のキュー・マネージャーを定義します。

これは、「必要なエラー・メッセージ」が選択されている場合のみ必要です。

エラー・キュー いいえ はい ERROR このパターン・パラメーターは、エラー・メッセージ用のキューを定義します。

これは、「必要なエラー・メッセージ」が選択されている場合のみ必要です。

正しくないメッセージ・アクション いいえ いいえ 保管 このパラメーターは、メッセージ・エレメントをルーティングできない場合に実行するアクションを定義します。 有効な値は以下のとおりです。
  • 保管
  • キャンセル

「保管」が選択されている場合、ルーティングできないメッセージ・エレメントは「正しくないメッセージ・キュー」に置かれます。

「キャンセル」が選択されていて、メッセージ・エレメントをルーティングできない場合、メッセージ・フロー全体がロールバックされます。

正しくないメッセージ・キュー・マネージャー いいえ はい   このパターン・パラメーターは、ルーティングできないメッセージ・エレメントを保存するためのキュー・マネージャーを定義します。 ブランクのままにすると、デフォルトはブローカー・キュー・マネージャーです。

このパラメーターが必要なのは、「正しくないメッセージ・アクション」パラメーターの値が「保管」の場合のみです。

正しくないメッセージ・キュー いいえ はい BAD このパターン・パラメーターは、ルーティングできないメッセージ・エレメントを保存するためのキューを定義します。

このパラメーターが必要なのは、「正しくないメッセージ・アクション」パラメーターの値が「保管」の場合のみです。

デフォルトが示されていない場合、デフォルト値は空ストリングです。
